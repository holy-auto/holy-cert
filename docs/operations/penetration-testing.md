# ペネトレーションテスト計画

外部ベンダーによる年 1 回 + 重大変更時の追加実施を基本とする。

## 1. スコープ

| 対象 | 内容 |
|---|---|
| Web app (`app.ledra.co.jp`) | 認証 / 認可 / OWASP Top 10 / API レベル攻撃 |
| Marketing (`ledra.co.jp`) | 情報漏洩 / DoS のみ (機能少ないため軽め) |
| API (`/api/*`) | 全エンドポイント、特に admin / webhook / cron |
| モバイル App | iOS / Android 各 1 回。reverse engineering 含む |
| インフラ | Vercel / Supabase / Upstash の設定不備 (限定的) |

## 2. 想定脅威モデル

| 攻撃者 | 想定能力 | 主な目標 |
|---|---|---|
| 外部スクリプトキディ | 公開ツール (sqlmap / Burp) | 不特定多数の脆弱サイトを自動探索 |
| 競合 / 内部告発志向 | アカウント保有 + 中級ペンテスタ | tenant escape, 課金回避, データ破壊 |
| 標的型攻撃者 | フィッシング + 0day | 個人情報取得, ランサムウェア |

## 3. 評価ベンダー候補(日本国内)

価格帯と特徴をまとめる。実際の発注は要件定義 + 3 社相見積もりで決める。

| ベンダー | 価格目安 | 強み |
|---|---|---|
| LAC | 200〜500万 | エンプラ向け実績多数, 報告書品質高 |
| 三井物産セキュアディレクション | 200〜400万 | SaaS 経験豊富 |
| イエラエセキュリティ | 150〜300万 | 価格と速度のバランス |
| GMO サイバーセキュリティ byイエラエ | 150〜350万 | リーズナブル, モバイルに強み |

## 4. 年間スケジュール

| 月 | 内容 |
|---|---|
| 1 月 | 前年テストの fix 完了確認 + 当年 RFP 起票 |
| 3 月 | ベンダー選定完了 |
| 5〜6 月 | 本番テスト実施 (4 週間枠) |
| 7 月 | 修正 sprint (高深刻度は 30 日 SLA) |
| 9 月 | re-test (修正確認) |
| 12 月 | 結果を SOC 2 監査資料 / 顧客向け summary に反映 |

## 5. 重大変更時の追加テスト

以下の変更があったらフルテストではなく対象限定テストを発注:

- 認証方式変更 (例: WebAuthn 投入)
- 決済処理変更 (例: 新決済プロバイダ追加)
- マルチテナント境界の変更 (例: テナント間共有機能の追加)
- 第三者 API 連携の追加 (新規 webhook 受信)

予算目安: 50〜150 万円 / 限定スコープ。

## 6. 内部での事前テスト (RFP 前)

ベンダー受注の品質を上げるため、社内で以下を実施:

- [ ] CodeQL extended scan の HIGH/CRITICAL を全クリア
- [ ] `npm audit --audit-level=high` をクリア
- [ ] Trivy fs scan で CRITICAL なし
- [ ] Burp Community でゴリラスキャン (fuzzer / spider) を実施
- [ ] `docs/operations/security-review-checklist.md` を全 admin ルートに当てる

## 7. 報告書の取り扱い

- ベンダー納品物は機微情報。**社外共有禁止**、1Password の `Confidential
  Reports` ボールトに保管
- 顧客向けには summary を作成 (発見件数 / 深刻度別 / 対応状況)
- 監査人 / 顧客のセキュリティ監査チームには NDA 締結の上で共有可
