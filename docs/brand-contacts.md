# インテグレーション・パートナー ロゴ使用許諾 問い合わせ先一覧

`src/components/marketing/IntegrationLogoWall.tsx` に掲載している 11 ブランドについて、ロゴ使用許諾を取得するための **問い合わせ先 / 公式ブランドキット URL / 主な使用条件** をまとめたもの。

> 現状コンポーネントはテキストチップ表示で運用しており、許諾が取れたパートナーから順次 SVG/PNG ロゴに差し替える方針（コンポーネント先頭コメント参照）。
>
> サンドボックス環境からは各社のページに直接 HTTP アクセスできないため、本ドキュメントの URL は Web 検索で得られた公開情報を元にしている。実際の問い合わせ前に各 URL の最新状態を必ずブラウザで確認すること。
>
> 各サービス名・ロゴ・商標はそれぞれの権利者に帰属する。

## 各社の使用条件と申請時の注意

### 1. Stripe
- **使用条件**: Stripe Marks Usage Terms に従う。ワードマークは slate / blurple / white の 3 色のみ。Stripe 連携製品では「Powered by Stripe」バッジを使うのが安全。提携や推奨を示唆する形での使用は別途書面許諾が必要。
- **問い合わせ先**: 商標関連は `trademarks@stripe.com`。
- **ブランドキット**: https://stripe.com/newsroom/brand-assets（SVG/PNG/EPS、透過背景あり）。
- **法的ルール**: https://stripe.com/legal/marks

### 2. Square（Block, Inc.）
- **使用条件**: Block, Inc. と認可済みライセンシーのみ広告・販促・販売用途に Square ロゴを使用可能。Square 決済を使う加盟店・開発者は「Square for Developers Trademark Terms」に基づき所定の素材を使用できる。
- **問い合わせ先**: 商標関連は `trademarks@squareup.com`。
- **メディアキット**: https://block.xyz/mediakit（Block / Square / Cash App など各ブランドキットへ）。
- **IP ポリシー**: https://squareup.com/us/en/legal/general/copyright

### 3. Google Calendar
- **使用条件**: Google Brand Resource Center の Trademark Rules に従う。改変・スペース確保（25px 以上）・名詞/動詞化禁止・形容詞としてのみ使用、など細則あり。提携の示唆は禁止。多くの用途で事前申請が必要。
- **問い合わせ先**: Brand Resource Center 内の許諾申請フォーム（about.google/brand-resource-center → "Submit a request"）。
- **ガイドライン**: https://about.google/brand-resource-center/guidance/
- **商標リスト**: https://about.google/brand-resource-center/trademark-list/

### 4. LINE（LINEヤフー株式会社）
- **使用条件**: LINEヤフーのブランド素材は原則として事前申請・許諾が必要。TVCM、大規模広告キャンペーン、提携プレスリリース等は事前許諾必須。一方で「LINE 公式アカウント」「LINE Mini App」「LINE ポイント」のサービスロゴについては、所定ガイドラインに準拠していれば事前申請不要のケースあり。

## ロゴ画像データの取得方針

サンドボックス環境では各社サイトへの直接ダウンロードが拒否された（`x-deny-reason: host_not_allowed`）ため、本コミットにはバイナリの取り込みを行っていない。下記いずれかの方法で取得する想定:

1. **許諾済みパートナー優先**: 上記コンタクトに使用許諾申請 → 許諾が下りたものから順に、各社の公式ブランドキット URL から SVG/PNG を取得し `public/brands/<vendor>/` 配下に配置する。
2. **配置場所の規約**: `public/brands/<vendor>/logo.svg`（ライト用）, `public/brands/<vendor>/logo-dark.svg`（ダーク用がある場合）, `public/brands/<vendor>/LICENSE.md`（使用条件・許諾日・連絡履歴）を必ずセットで置く。
3. **コンポーネントへの反映**: `src/components/marketing/IntegrationLogoWall.tsx` の `INTEGRATIONS` 配列に `logo` フィールドを追加し、許諾済みのものから `<Image>` または `<img>` でレンダリング、未許諾のものはテキストチップのままにする。
4. **削除トリガー**: 許諾が取り消された／条件変更があった場合は `public/brands/<vendor>/` ごと削除し、コンポーネントもテキスト表示に戻す。

——
株式会社●●
●●（担当者名）
