## 0. 結論 (一行サマリ)

> **「整備 5.5 兆円市場の "履歴" レイヤを取りに行く施工店オーナー出身の Founder」** を一言で刺し、**希望金額 ¥3,000 万・使途は損保ジャパン本番接続コストと初期 100 店舗キャンペーン補填** で押す。**No Money でも事業は止めない、放送そのものが GTM** と最初に宣言してプレッシャーを反転させる。

## 1. オープニング原稿 (暗記必須・30 秒)

> 番組冒頭、虎の前に出た瞬間に言い切る部分。**ここで掴めなければ後半の数字は聞かれない**。

```
株式会社HOLY 代表取締役の堀越友輔です。
本日は ¥3,000 万円のご融資をお願いに参りました。

私は元々、自動車のコーティング・PPF 施工店のオーナーです。
真面目に施工する事業者ほど、紙とLINEとExcelで消耗している。
一方で保険会社や中古車市場は、「いつ・誰が・何を施工したか」を
後から検証する手段を持っていません。

私はこの「車に施工した事実」を、デジタル証明書として
発行・流通させるプラットフォーム『Ledra』を運営しています。
施工店向け業務 SaaS + 施工証明書ネットワークの一体型です。

本日いただく ¥3,000 万円は、確定した損保ジャパンとの
本番接続コスト、および先着 100 店舗キャンペーンの補填に
全額充当します。よろしくお願いいたします。
```

ポイント:
- **3 秒以内に金額を出す**: 「お願いに参りました」の後に間を置かない
- **元施工店オーナー** の一次情報フックを冒頭に置く (現場経験 → 開発の物語)
- **損保ジャパン** は実名で出して具体性を担保 (確定済の 2026-05-13 経営判断)
- **使途を一文で**: 「全額充当」と言い切る。曖昧にしない

## 6. 想定質疑応答

虎の典型的な攻撃パターンを 4 系統に分類し、それぞれの切り返しを用意する。

### 6.1. 論理派・数字派 (例: 林尚弘型 / 桑田龍征型)

「で、儲かるの?」「Unit Economics は?」を 30 秒で詰めてくるタイプ。

| 質問 | 回答 |
|---|---|
| **CAC と LTV は?** | 「現状は直販中心で CAC が読みづらいですが、加盟店オンボーディングコスト 1 件あたり実費約 ¥XX 万、LTV は平均契約期間 XX ヶ月 × ARPU ¥35,000 で ¥XX 万。LTV/CAC は X.X 倍を狙っています。CAC payback は 12 ヶ月以内が目標です」 |
| **月次バーンレートは?** | 「現状 ¥XX 万。融資後は採用込みで ¥XX 万に上がる想定です。12 ヶ月分のランウェイを確保した上で攻めます」 |
| **粗利率は?** | 「サブスク部分は約 80%。Polygon ガス費とサポート工数を引いた実効粗利は 65〜70%。ハードウェアを売らないので、SaaS としては標準的な数字です」 |
| **店舗数 2,400 で ARR 10 億は本当に届くのか?** | 「店舗単独では届きません。だから 3 階建てです。店舗 ¥6 億 + 保険会社 ¥3 億 + OEM/流通 ¥1 億 で 10 億。**保険会社 1 社で月 ¥500-1,000 万取れる前提**は、損保ジャパンとの先方提示で既に出ている数字です」 |
| **そもそも整備店は SaaS に金払うのか?** | 「払う事業者は払う、というのが現状の答えです。フリープランで月 10 件まで無料、有料転換率は現状 XX% です。**真面目な事業者ほど書類で消耗している** のが現場の事実で、その層が初期顧客になります」 |

### 6.2. 人物深掘り型 (例: 加藤浩次型 / 林貴志型)

「お前は何者なんだ?」「覚悟は?」「家族は?」を聞いてくるタイプ。**沈黙が一番怖い。3 秒以内に答え始める**。

| 質問 | 回答 |
|---|---|
| **元々何やってた?** | 「自動車のコーティング・PPF 施工店 detailing_holy をやっていました。今も運営は続けています。**現場の不便を、自分で書いて自分で売っている** プロダクトです」 |
| **これダメだったらどうする?** | 「detailing_holy の現場に戻ります。ただ、Ledra は止めません。**プロダクトは既にあって、毎日動いています**。融資が無くても、規模が小さくなるだけで、続けます」 |
| **借金は?** | 「会社の借入は ¥XX 万。個人保証は無し。**個人の借入はゼロ** です」 (←要事前確認・嘘は絶対 NG) |
| **家族の理解は?** | 「妻 / 家族には事業内容と覚悟を説明しています。**起業 1 年目は家計を圧迫しましたが、2 年目以降は安定した役員報酬を取れる状態** です」 (←要事前確認) |
| **なぜ君がやる? 他の人でもできるじゃないか** | 「**施工店オーナーで、かつ SaaS を自分で書ける人間**は国内にほぼ存在しません。私は両方をやっています。現場ヒアリング → 翌日に機能をリリース、のサイクルを内製で回せるのが差別化です」 |
| **5 年後どうなってる?** | 「**車の施工履歴の標準インフラ** になっています。中古車を買うときに『この車、Ledra で証明書が出ているか』を消費者が確認する状態。これがゴールです」 |

### 6.3. 競合・市場懐疑型 (例: 岩井良明型 / その他テック系虎)

「もう似たやつあるんじゃないの?」「OEM が自分でやらない?」を聞くタイプ。

| 質問 | 回答 |
|---|---|
| **競合は?** | 「業務管理 SaaS は紙 + Excel が国内のほぼ全部。Shopmonkey・Tekmetric は米国専業で日本未上陸。証明書発行は店舗ごとの自前 PDF。**業務 + 証明書 + 流通の 3 つを統合する事業者は国内に存在しません**」 |
| **トヨタが自分でやるんじゃないの?** | 「やる可能性はあります。ただ、**OEM が単独で業界標準を作るのは、紙の整備手帳の歴史を見れば 30 年かかる**。我々のような業界横断のプレイヤーが先に取って、OEM が乗ってくる方が早い。Woven by Toyota の Partner Program はその文脈で進めています」 |
| **保険会社が自分で証明書発行すれば?** | 「保険会社は施工しないので、**施工現場とのインターフェースが無い**。施工店 → 保険会社の流れを作るには、施工店向け SaaS を持っている必要があり、ここが我々の入口です」 |
| **ブロックチェーンって本当に必要?** | 「証明書の改ざん検知の手段として、現実的に Polygon が最も安く運用できるからです。**ブロックチェーンを売っているのではなく、「5 年後も検証できる証明書」を売っている**。手段としての Polygon です」 |

### 6.5. 過去失敗・スキャンダル型 (一番厄介)

「Twitter で前にこういうこと言ってたよね?」「過去に倒産させた?」を放送前に調べてくる。

**事前にやっておくこと**:
1. 自分の X (旧 Twitter) 過去ツイートを 3 年分遡って確認、削除すべきは削除
2. 過去の会社の登記簿を取得し、未払金 / 破産履歴がないか確認
3. SNS で過去のお客様トラブルが残っていないか検索 (`detailing_holy トラブル` 等)
4. ググラビリティ確認: 「堀越友輔 評判」「ledra 評判」

**万が一突かれた時**:
- **絶対に嘘をつかない**。「事実です。当時はこういう状況で、こう判断しました。今は反省して XX しています」と短く認める
- 言い訳・他責は最悪。「自分の責任です」で始める
- 30 秒以上引きずらない。「次に行きましょう」と司会に促してもらう
