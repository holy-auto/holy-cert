## 3. 業界の課題 (掴み 3 枚)

### 3.1 施工店の現場

- 施工記録は **紙の作業伝票 + LINE 写真 + Excel 顧客台帳** に分散
- 1 件あたり書類対応に **20〜40 分** / 月で店舗あたり **30〜60 時間** の工数ロス
- 顧客から「3 年前に何のコーティングをしてもらった?」と聞かれて、出てこない
- スタッフが辞めると顧客台帳ごとブラックボックス化

> **真面目に施工している事業者ほど、書類で消耗する。**

### 3.2 保険会社の現場

- 不正コーティング請求・水増し請求の確認手段が乏しい
- 施工店ごとに様式バラバラの PDF を、目視で確認するしかない
- 業界全体で年間 **数百億円規模** の損保支払いに不透明性が残る

### 3.3 中古車流通の現場

- 「この車にコーティング・PPF・板金歴があるか」を買い手が検証できない
- 査定が下振れする (正当な施工歴が値段に乗らない) / 上振れする (偽申告)
- UCDA など業界団体は透明化を進めようとしているが、施工レイヤだけ空白

## 4. Ledra の解決策

### 4.1 プロダクト概要

**4 ポータル + モバイルアプリ** で、施工店の業務 SaaS と
業界横断の施工証明書ネットワークを **地続きの 1 プロダクト** にしています。

| ポータル | ユーザー | 役割 |
|---|---|---|
| Admin | 施工店 | 予約 → 来店 → 作業 → 証明書発行 → 請求 → 入金 を 1 画面で完結 |
| Agent | 代理店 | 施工店紹介 / コミッション管理 |
| Insurer | 保険会社 | 証明書検索 / 案件管理 / 真正性検証 |
| Customer | 顧客 | QR からスマホで自分の証明書を閲覧 |
|  | + モバイル | iOS / Android で現場運用 (Tap to Pay / NFC / 車検証 OCR) |

### 4.2 施工証明書の "技術的な硬さ"

- **公開検証**: `https://ledra.co.jp/c/[public_id]` で誰でも検証可能
- **Polygon ブロックチェーンアンカリング**: 施工写真の SHA-256 をオンチェーン記録 → 後日改ざん不能
- **電子署名**: 顧客の受領サイン (`/sign/[token]`) で「受け取った」事実を法的レベルで保管
- **NFC 連携**: 車両に貼ったタグから、その車の施工履歴に直接遷移

### 4.3 なぜ "二重ネットワーク効果" が成立するか

```
施工店が業務 SaaS として使う → 証明書が自然と溜まる
                                ↓
        証明書を参照する保険会社・中古車市場が増える
                                ↓
   「Ledra 証明書を出してくれる店舗」を保険会社・中古車事業者が指名
                                ↓
        指名されたい施工店が Ledra 加盟 (= 新規施工店流入)
                                ↓
                     さらに証明書が溜まる (ループ)
```

業務 SaaS の **ユーティリティ価値** と、施工台帳の **流通価値** が
別々に回り、後者が前者を引き上げる構造です。

## 5. なぜ今 (Why Now)

- **SDV (Software Defined Vehicle) 化** で車のメンテ履歴のデジタル化要請が加速
- **2024 年 OBD 検査開始** → 整備記録の電子化が国側からも要請
- **損保業界の不正請求対策強化** (修理費水増し・コーティング水増しが社会問題化)
- **中古車流通の透明性ニーズ** (UCDA / J.D. Power) が高まり、施工歴の可視化に需要

> **「車の履歴がデジタル化される 10 年に 1 度のタイミング」** に立っています。

## 11. 虎が突っ込むであろう論点 (先回り FAQ)

| 想定質問 | 回答骨子 |
|---|---|
| 「もう似た会社あるんじゃないの?」 | 整備 SaaS は存在するが、コーティング・PPF・板金など "施工" 領域 + 流通レイヤを統合した事業者は国内に存在しない。競合マップで空白を可視化済 |
| 「結局、施工店から月 ¥24,800 もらうだけのビジネスじゃないの?」 | サブスクは入口。本命は保険会社・中古車市場からの **流通課金**。施工店ネットワーク自体が交渉カードになる |
| 「Founder は元施工店オーナー。技術は誰が?」 | フルスタックチームを社内に保有。Next.js 16 + Supabase + Stripe + Polygon の本番運用実績あり。240+ API / 2,000+ テストが内製の証拠 |
| 「保険会社が本当に金を払うのか?」 | 現在 PoC 協議中の保険会社 X 社。不正請求対策 + 案件処理工数削減の ROI で握れる見立て |
| 「ブロックチェーンって本当に必要?」 | 必須ではないが、保険会社・中古車流通に "改ざんされていない" 証跡を提示する上で **第三者検証可能性** が決定的に効く。コストは Polygon で月数千円レベル |
| 「ALL or NOTHING の金額の根拠は?」 | 12 ヶ月で加盟店舗 N2 + 保険会社 1 社本番接続まで持っていくのに必要な人件費 + 認証取得 + 営業費用から逆算 |
