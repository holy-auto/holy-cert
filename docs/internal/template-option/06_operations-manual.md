# オペレーション担当者向け運用手順書

> CARTRUST テンプレートオプションの日常運用手順。
> 対象: 制作担当者・カスタマーサポート・運営管理者

---

## 1. 運用体制

### 想定人員
- **制作担当**: 1名（月2〜5件のB制作 + 追加修正作業）
- **カスタマーサポート**: 兼務可（A契約はセルフサービスのためサポート問い合わせ少）

### 使用画面
- オーダー管理: `/admin/platform/template-orders`
- オーダー詳細: `/admin/platform/template-orders/[id]`
- テンプレート管理: `/admin/platform/templates`
- 契約一覧: `/admin/platform/template-subscriptions`

---

## 2. A（ブランド証明書 ライト）の運用

### フロー概要
A はセルフサービスのため、運営側の作業は基本的に発生しない。

```
[加盟店] 申込・決済 → テンプレ選択 → 設定入力 → プレビュー → テスト発行 → 利用開始
         ↓ (問い合わせ時のみ)
[運営]   サポート対応
```

### 運営側で対応が必要なケース

| ケース | 対応 |
|---|---|
| 設定方法の問い合わせ | ヘルプページ誘導 or 画面共有サポート |
| ロゴがうまく表示されない | ファイル形式・サイズ確認 → 加工代行は追加作業費 |
| 「レイアウトを変えたい」 | B（プレミアム）へのアップグレード or 追加作業を案内 |
| 解約したい | Stripe Portal誘導。データは30日保持の旨を説明 |
| 支払い遅延 | Stripeが自動リトライ。3回失敗後にsuspended |

## 3. B（ブランド証明書 プレミアム）の運用

### 全体フロー

```
Step 1. 申込受付（自動）
  └→ Stripe決済完了 → Slack通知 → オーダー管理画面に表示

Step 2. ヒアリング確認（1営業日以内）
  └→ 提出素材・ヒアリング内容を確認
  └→ 不足があればテナントにメッセージ
  └→ ステータス: paid → hearing

Step 3. 制作着手（5営業日）
  └→ ステータス: hearing → in_production
  └→ config_json作成 + layout調整
  └→ プレビュー生成

Step 4. 初稿レビュー（3営業日）
  └→ ステータス: in_production → review
  └→ テナントにプレビュー送付（メール + 管理画面）
  └→ テナントがレビュー・FB

Step 5. 修正（3営業日）
  └→ ステータス: review → revision → review
  └→ 修正回数カウント（max_revisions以内か確認）
  └→ 超過時は追加作業費を案内

Step 6. テスト発行
  └→ ステータス: review → test_issued
  └→ テナントにテスト発行済みの証明書を確認依頼

Step 7. 承認・公開
  └→ テナント承認 → ステータス: approved → active
  └→ tenant_template_configs.status = active
  └→ tenant_option_subscriptions 作成（月額課金開始）
  └→ Stripe Subscription 作成（¥4,400/月）
```

### ステップ別 作業チェックリスト

#### Step 2: ヒアリング確認
- [ ] ロゴデータの確認（形式・解像度・背景透過）
- [ ] ブランドカラーの確認（HEXコード取得）
- [ ] 保証文言の確認（テキスト内容の読み合わせ）
- [ ] 注意文言の確認
- [ ] メンテナンスURLの動作確認
- [ ] 追加要望の確認・実現可否判断
- [ ] 不明点があればテナントにメッセージ送信
- [ ] ステータスを `hearing` に変更
- [ ] 対応履歴にコメント記入

#### Step 3: 制作
- [ ] tenant_template_configs レコード作成（option_type=custom）
- [ ] config_json の作成（ヒアリング内容を反映）
- [ ] layout_key の決定
- [ ] ロゴの配置確認
- [ ] カラーの反映確認
- [ ] 保証文言・注意文言の反映確認
- [ ] メンテナンスURL・QRの反映確認
- [ ] PDF出力テスト
- [ ] ステータスを `in_production` に変更

#### Step 4: レビュー
- [ ] プレビューPDFを生成
- [ ] テナントにレビュー依頼メッセージを送信
  - メッセージ例: 「初稿が完成しました。管理画面のオーダー詳細からプレビューをご確認ください。修正点がありましたらメッセージでお知らせください。」
- [ ] ステータスを `review` に変更

#### Step 5: 修正
- [ ] テナントからのFB内容を確認
- [ ] 修正可能範囲か判断（CARTRUST構造を逸脱しないか）
- [ ] 修正不可の場合は理由を説明
- [ ] config_json を更新
- [ ] プレビュー再生成
- [ ] revision_count をインクリメント
- [ ] max_revisions 超過チェック → 超過時は追加作業費を案内
- [ ] ステータスを `revision` → 修正完了後 `review` に変更

#### Step 6-7: テスト発行・公開
- [ ] テスト発行実施
- [ ] テナントに確認依頼
- [ ] テナント承認確認
- [ ] tenant_template_configs.status = active, is_default = true
- [ ] tenant_template_configs.published_at = now()
- [ ] Stripe Subscription 作成（月額¥4,400）
- [ ] tenant_option_subscriptions レコード作成
- [ ] ステータスを `active` に変更
- [ ] 完了メッセージ送信

## 4. 追加作業の運用

### フロー
```
[加盟店] 追加作業依頼フォーム送信
  ↓
[運営]  内容確認 → 見積金額決定 → Stripe Invoice発行
  ↓
[加盟店] Invoice支払い
  ↓
[運営]  作業実施 → 確認依頼 → 反映
```

### 見積基準

| 作業種別 | 基準金額 | 目安工数 | 判断基準 |
|---|---|---|---|
| 文言修正 | ¥5,500 | 0.5h | テキスト差替のみ |
| 文言修正（複数箇所） | ¥8,800 | 1h | 3箇所以上 |
| レイアウト調整（軽微） | ¥11,000 | 1h | 配置の微調整 |
| レイアウト調整（中規模） | ¥22,000 | 2h | セクション追加・構成変更 |
| QR/URL差し替え | ¥3,300 | 0.5h | URL・ラベル変更のみ |
| テンプレート追加制作（既製ベース） | ¥33,000 | 3h | 既製テンプレ+カスタム |
| テンプレート追加制作（オリジナル） | ¥55,000 | 5h | フルカスタム |
| 大幅再設計 | 別見積 | 要ヒアリング | レイアウト全面変更 |

### Stripe Invoice 発行手順
1. A3画面 → 「追加作業を登録」ボタン
2. 作業内容・金額を入力
3. 「Invoice発行」ボタン → Stripe APIでInvoice作成・送信
4. テナントにメール通知（Stripe自動）
5. 支払い確認後、作業開始

## 5. 日常チェック項目

### 毎朝（5分）
- [ ] オーダー管理画面で未対応オーダーを確認
- [ ] `paid` ステータスのオーダーがないか（着手漏れ防止）
- [ ] `review` ステータスで3営業日以上経過しているものがないか（テナント催促）

### 週次（15分）
- [ ] 契約一覧で `past_due` のテナントがないか確認
- [ ] 進行中オーダーの納期確認
- [ ] テンプレートギャラリーの表示確認（サムネ壊れ等）

### 月次（30分）
- [ ] MRR確認（A/B別）
- [ ] 新規契約数・解約数の集計
- [ ] 追加作業収益の集計
- [ ] よくある問い合わせ内容の整理

## 6. エスカレーション基準

| 状況 | エスカレーション先 | 対応 |
|---|---|---|
| テナントが「法的に有効な証明書にしたい」と要求 | PM/経営 | 「弁護士レビュー不含」を再説明。CARTRUST側の法的責任は負わない旨を伝達 |
| 「自由にデザインしたい」と要求 | PM | B契約の範囲を説明。逸脱する場合は別見積 |
| テンプレートがPDF出力で崩れる | 開発 | layout_key / config_json の不具合調査 |
| Stripe課金トラブル | 開発 | Stripe Dashboard確認 + 必要に応じて手動処理 |
| 修正が3回以上往復 | PM | 要件の再確認。追加作業費の発生を加盟店に説明 |

## 7. テンプレート用語集（社内統一）

| 用語 | 意味 |
|---|---|
| 既製テンプレ | platform_templates に登録されたテンプレート |
| テンプレ設定 | tenant_template_configs のレコード |
| config_json | テンプレート設定のJSONデータ |
| layout_key | PDFレイアウトの識別子（standard/premium等） |
| テスト発行 | サンプルデータで証明書PDFを生成する機能 |
| 構造化カスタム | config_json のスキーマ内での変更。自由編集ではない |

## 8. よくある質問（社内FAQ）

**Q. テナントから「ロゴの背景を透過にしてほしい」と言われた**
A. 画像加工は追加作業費（¥5,500〜）。テナントに透過PNGの再提出を依頼するか、費用をご案内してください。

**Q. テナントが保証文言を500文字以上書きたいと言っている**
A. スキーマ上限は500文字。超過分はカスタムセクション（B契約なら3つまで追加可能）で対応可能。

**Q. テナントから「A4じゃなくB5にしたい」と言われた**
A. 現状A4のみ対応。用紙サイズ変更は将来検討だが現時点では不可。

**Q. テナントが解約したが「テンプレートデータを返してほしい」と言っている**
A. config_jsonのエクスポートは可能（JSON形式）。ただしCARTRUST上でのみ動作するため、他サービスでの利用は不可。

**Q. テスト発行で実際の顧客データが使われないか心配**
A. テスト発行はサンプルデータ（ダミー）を使用。実顧客データは使用しません。
