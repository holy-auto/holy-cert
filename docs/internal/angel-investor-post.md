# Ledra — エンジェル投資家アプローチ用 Founder 投稿文

> 用途: Founder (堀越) 名義で**エンジェル投資家**に向けて出す投稿文 / DM / メール
> 提出会社: 株式会社HOLY ／ サービス名: Ledra (旧: holy-cert)
> 想定チャネル: X (旧Twitter) ・note・エンジェル投資家マッチング/個人投資家への直接連絡
> 最終更新: 2026-05-18
> 兄弟資料: `docs/internal/vc-submission.md` (VC ファンド向け) / `docs/internal/theta-times-ventures-inquiry.md` / `docs/internal/csp-fund-inquiry.md` / `docs/internal/vc-pitch-outline.md`

VC ファンド向けパッケージ (兄弟資料) を**そのまま流用しない**こと。エンジェルは
性格が違う。本資料は「**Founder 個人の一次情報と覚悟**」を主役に据える。

## 0. 事前に押さえる前提 (社内向けメモ・投稿文には書かない)

| 項目 | エンジェル投資家 (個人) |
|---|---|
| 資金の出どころ | **自分の金**。ファンドの LP 説明責任が無い分、判断が速く、人で決める |
| 小切手サイズ | 数十万〜数千万円 / 1 名。複数名で**エンジェルラウンド**を組むのが前提 |
| 重視する点 | ① Founder の現場一次情報と本気度 ② プロダクトが既に動いているか (リスク低減) ③ なぜ今・なぜあなたか ④ 自分の経験/人脈が効くか |
| ファンドとの差 | テーゼ適合より**「この人に賭けたい」**が先。投資後は資金より**人脈・最初の信用・採用・壁打ち**で効く |
| 日本特有 | **エンジェル税制** / **J-KISS** で条件を簡素化できると意思決定が速い (適格性は要確認) |

**ポジショニング判断 (重要)**: VC 版は「事業の構造・市場規模・テーゼ適合」が
主役。エンジェル版は **「施工現場出身の Founder が、自分で痛みを引き受けて
プロダクトを本番まで作り切った」物語**を主役にする。数字は盛らず、出ている
ものだけを出す。プロダクトが既にあること = リスクが低いことを淡々と示す。

ブランドトーン (兄弟資料 §補足 / `docs/marketing/case-study-guide.md` と統一):
- 詩的すぎず実務的すぎず / **数字で語る**
- やらないこと: ベンダーロック表現 / 他社批判 / **数字の盛り**

## 2. メイン投稿文 (note / 長文ポスト / 個人投資家への初回連絡本文)

> Founder 一人称。約 1,100 字。エンジェルが最初に読む想定。
> 物語 → プロダクトの現実 → 数字 → 何に賭けているか → お願い、の順。

```
はじめまして。株式会社HOLY 代表取締役の堀越 友輔と申します。
自動車のディテイリング (detailing_holy) を自分の手でやってきた人間です。

現場でずっと引っかかっていたことがあります。
1 台に何時間もかけてコーティングや PPF を施工しても、その「施工した事実」が
紙の証明書か、LINE の写真か、Excel の台帳にしか残らない。顧客が証明書を
失くす。数年後に保険会社や中古車買取が「本当に施工したのか」を確認したくても、
誰も検証できない。真面目に施工している事業者ほど、書類と確認の問い合わせに
時間を削られていく。これは自分の店の話であり、業界全体の話でした。

この課題を、外注ではなく自分たちで作り切ったのが「Ledra」です。

Ledra は、施工店向けの業務 SaaS (予約 → 来店 → 作業 → 証明書発行 → 請求 →
入金を 1 画面に統合した案件ワークスペース) を入口に、発行した施工証明書を
顧客・保険会社・中古車市場が共通で参照できる "車の施工台帳" に育てる
マルチテナント SaaS です。証明書は QR / URL で誰でも公開検証でき、写真の
ハッシュは Polygon にアンカリングして第三者の改ざんを検知します。施工店
(Admin)・代理店 (Agent)・保険会社 (Insurer)・顧客 (Customer) の 4 ポータルと
モバイルアプリ (iOS / Android) を提供しています。

ピッチではなく事実だけ書きます。プロダクトは構想段階ではなく本番稼働済みで、
130+ の DB マイグレーション、400+ の API、2,000+ のテストケースが動いています。
対象は国内約 5.5 兆円・数万事業者という、紙と Excel と LINE で回り続けてきた
レガシー産業です。今、損害保険会社との PoC を進めています。

賭けているのは「施工店 SaaS」単体ではありません。施工の履歴が業界横断で
検証できるようになると、保険会社・中古車流通・OEM が証明書を参照し、上流
からの逆引きで施工店が Ledra を採用する — 業務 SaaS と証明書ネットワークが
地続きの二重ネットワーク効果です。技術はもう自分たちの手の中にあります。
次の山は、資金と、保険会社・OEM への到達速度、そして採用です。

そこで、エンジェル投資家の方にお願いがあります。資金はもちろんですが、
それ以上に、整備・保険・自動車流通のリアルな人脈、エンタープライズ商談の
壁打ち、最初の信用を貸していただける方と組みたいです。プロダクトはあります。
論点は「作れるか」ではなく「どこまで速く広げられるか」です。

サービス紹介資料・5 分の紹介動画・デモ環境はすぐに共有できます。
30 分だけ、お話しさせていただけませんか。

株式会社HOLY 代表取締役 堀越 友輔
LP: https://ledra.co.jp ／ 公開検証: https://ledra.co.jp/verify
紹介動画: https://ledra.co.jp/video ／ X: https://x.com/detailing_holy
連絡先: <メール: ◯◯@ledra.co.jp> ／ <電話>
```

## 6. エンジェルに特化した訴求ポイント (面談・本文で必ず握る)

VC は市場とテーゼ。エンジェルは**人と、自分が出せる価値**で決める。
以下を投稿文・面談の軸にする。

```
1. なぜ自分か (Unfair Advantage)
   施工現場を自分で回してきた一次情報。業界の痛みを "聞いた" のではなく
   "受けた" 側。現場ヒアリング→機能化を内製で回せる。

2. なぜ今か (Why Now・短く)
   SDV 化・車検/OBD 電子化・損保の不正請求対策強化・中古車流通の透明性要求。
   "車の履歴がデジタル化される 10 年に 1 度" にレガシー産業側から入る。

3. リスクは既に下げてある
   構想段階の調達ではない。本番稼働済みプロダクト + 損保 PoC 進行中。
   論点は「作れるか」ではなく「どこまで速く広げるか」。

4. エンジェルに具体的に求めること (資金以外)
   - 整備 / 損害保険 / 中古車流通 / OEM への人脈・紹介
   - エンタープライズ商談 (長期・大企業) の壁打ちと同席
   - 採用 (BizDev / CS コア) の初期母集団
   - 「最初の 1 社」を口添えできる信用

5. 条件は簡素に
   エンジェルラウンド想定。<J-KISS / 普通株 など想定スキーム>。
   <エンジェル税制適格の有無は要確認のうえ、適格なら明記すると意思決定が速い>。
```

## 8. 盛らない・埋める箇所 (送信前に必ず確認)

- `<メール>` `<電話>` `<年齢>` `<学歴>` `<職歴>` を本人が記入
- トラクション実数 (導入店舗数 / 月間証明書発行数 / MRR) は**この投稿文に
  数値を書いていない**。書く場合は実値のみ。見込み・希望値を断定形で書かない
- 「損保 PoC 進行中」は事実状態に合わせる。接点フェーズが PoC 前なら
  「損害保険会社へのアプローチを開始」等、実態に合った表現へ差し替える
  (根拠: `docs/ledra-goals-strategy-2026-05.md` の Lighthouse 戦略・現フェーズ)
- 「本番稼働済み」「130+ migration / 400+ API / 2,000+ tests」は
  `README.md` / `FEATURES.md` と整合。数字が変わったら追従する
- スキーム (J-KISS / 普通株) ・エンジェル税制適格性は経営判断で確定してから明記
