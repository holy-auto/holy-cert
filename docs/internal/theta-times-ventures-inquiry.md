## 2. サービス説明 (本文・約520字 / フォーム自由記述欄想定)

> ※ CSP 版と本文は共通だが、Theta Times には冒頭で "5.5兆円レガシー産業の
> 履歴レイヤ" のフレーミングを必ず添える (§4 と接続させる)。

```
Ledra は、自動車のコーティング・PPF・板金・整備など「車に施工した事実」を
デジタル証明書として発行・流通させるマルチテナント SaaS です。対象は国内
約5.5兆円規模・数万事業者の自動車整備/カーケア領域 — 紙とExcelとLINEで
回り続けてきた典型的なレガシー産業です。

施工店の現場では、施工記録・顧客情報・車両情報・保険会社向けの確認情報が
分断され、真面目に施工する事業者ほど書類・確認工数に時間を取られています。
一方で保険会社や中古車流通の現場では「いつ・誰が・どんな施工をしたか」を
後から検証する手段が乏しく、不正請求や品質トラブルの温床になっています。

Ledra は施工店向けの業務 SaaS (予約 → 来店 → 作業 → 証明書発行 → 請求 →
入金を1画面に統合した案件ワークスペース) を入口に、発行された施工証明書を
顧客・保険会社・中古車市場が共通参照できる "車の施工台帳" に育てています。
証明書は QR / URL で公開検証でき、写真ハッシュは Polygon にアンカリング
することで第三者改ざん検知を実現しています。

施工店 (Admin) / 代理店 (Agent) / 保険会社 (Insurer) / 顧客 (Customer) の
4ポータル + モバイルアプリ (iOS / Android) を提供し、レガシー産業に分散した
価値の流れを1つのプラットフォームへ集約することを目指しています。
```

## 4. ★ ストーリーライン — "リアル産業の履歴レイヤ" (本資料の核)

Theta Times は「リアル産業 (製造・建設・物流・小売・ヘルスケア、約700兆円)
のレガシー DX」に賭けるファンド。**Ledra を "店舗 SaaS" として説明すると
刺さらない。"レガシー産業の履歴インフラ" として位置づける**。
`docs/internal/vc-pitch-outline.md` §10「整備5.5兆円市場の"履歴"レイヤを
取りに行く」と完全に揃えること。

### 4-1. テーゼ接続の1段論法 (面談冒頭で握らせる順序)

```
1. 対象は国内約5.5兆円・数万事業者の自動車整備/カーケア
   — 紙・Excel・LINE で回り続けた、御社の言う「リアル産業」そのもの
2. Ledra は単なる店舗 SaaS ではなく、施工の「履歴」を業界横断で検証可能に
   する "履歴レイヤ" (証明書ネットワーク)
3. 履歴レイヤは保険会社・中古車流通・OEM が参照する → 上流からの逆引きで
   施工店が Ledra を採用する二重ネットワーク効果
4. 結果、レガシー産業の取引データが Ledra に集約される = リアル産業 DX の
   インフラポジション
```

### 4-2. Why Now (Theta Times が重視する "今やる必然性")

```
- SDV (Software Defined Vehicle) 化と整備履歴デジタル化の制度的潮流
- 車検 / OBD 検査の電子化スケジュール
- 損保業界の不正請求対策強化 (施工事実の検証ニーズが顕在化)
- 中古車流通の透明性ニーズ (査定の根拠データ要求)
→ "車の履歴がデジタル化される10年に1度のタイミング" にレガシー産業側から入る
```

### 4-3. なぜ我々が勝てるか (Founder の一次情報 + 既存資産)

```
- 自動車ディテイリング事業 (detailing_holy) 出身の Founder = 施工現場の
  一次情報を持ち、現場ヒアリング→機能化を内製で回せる
- プロダクトは仮説段階ではなく本番稼働済 (130+ migration / 400+ API /
  2,000+ tests) = "作る" ではなく "伸ばす" フェーズ
- 業務 SaaS と証明書ネットワークが地続きの1プロダクト = 模倣困難
```
