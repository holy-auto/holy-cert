## 2. サービス説明

> フォーム文字数の感覚: 400〜600 字程度が読みやすい。以下は約 520 字版。
> もう少し短くしたい場合は §2-Short を使ってください。

```
Ledra は、自動車のコーティング・PPF・板金・整備など「車に施工した事実」を
デジタル証明書として発行・流通させるマルチテナント SaaS です。

施工店の現場では、施工記録・顧客情報・車両情報・保険会社向けの確認情報が
分断されており、真面目に施工している事業者ほど書類対応や確認工数に時間を
取られています。一方で保険会社や中古車流通の現場では「いつ・誰が・どんな
施工をしたか」を後から検証する手段が乏しく、不正請求や品質トラブルの温床に
なっています。

Ledra は、施工店向けの業務 SaaS (予約 → 来店 → 作業 → 証明書発行 → 請求 →
入金を 1 画面に統合した案件ワークスペース) を入口に、発行された施工証明書を
顧客・保険会社・中古車市場が共通参照できる "車の施工台帳" に育てています。
証明書は QR / URL で公開検証でき、写真ハッシュは Polygon にアンカリング
することで第三者改ざん検知を実現しています。

施工店 (Admin) / 代理店 (Agent) / 保険会社 (Insurer) / 顧客 (Customer) の
4 ポータル + モバイルアプリ (iOS / Android) を提供し、業界に分散していた
価値の流れを 1 つのプラットフォームに集約することを目指しています。
```

### 2-Short. 短縮版 (約 240 字)

```
Ledra は、自動車のコーティング・PPF・板金・整備など「車に施工した事実」を
デジタル証明書として発行・流通させるマルチテナント SaaS です。

共通参照できる "車の施工台帳" に育てます。証明書は QR / URL で公開検証でき、
写真ハッシュは Polygon にアンカリングすることで改ざん検知を担保しています。
施工店 / 代理店 / 保険会社 / 顧客の 4 ポータル + モバイルアプリを提供。
```

### 2-Talking-Points. 面談時に口頭で補足する論点

- **市場**: 国内の自動車整備・コーティング・PPF・板金市場は 5.5 兆円規模。
  施工店は数万社あり、ほぼ全店舗が紙 + LINE + Excel で施工記録を管理。
- **エッジ**: 業務 SaaS としての価値 (店舗 DX) と、施工台帳としての価値
  (保険・中古車流通) の **二重ネットワーク効果** が成立しうる構造。
- **テクノロジー**: Next.js 16 + Supabase + Stripe + Polygon アンカリング。
  証明書改ざん検知と RLS によるテナント分離を初期から内蔵。
- **競合**: 業務管理は紙 + Excel が大半。証明書発行は店舗ごとの自前 PDF。
  両方を統合し、かつ流通レイヤを持つ事業者は国内に存在しない認識。

### 5-1. 添付用 1 枚要約 (PDF 化推奨)

```
■ サービス名      Ledra (https://ledra.co.jp)
■ 運営会社        株式会社HOLY (代表取締役 堀越 友輔)
■ 設立            20XX 年 / 本社: 東京都港区

                  1 つのプラットフォームに統合。

■ 課題 (Why Now)
  - 自動車施工の記録は紙 + LINE + Excel が主流で、紛失・改ざん耐性が無い
  - 保険会社・中古車流通は "いつ誰が何を施工したか" を検証する手段が無い
  - 法令・SDV 対応で「車のメンテ履歴のデジタル化」要請が急速に高まっている

■ 解決策
  - 施工店向け業務 SaaS (予約→作業→証明書→請求を 1 画面で完結)
  - QR / URL でアクセス可能なデジタル施工証明書 (公開検証)
  - 保険会社・中古車市場が横断で参照できる施工台帳レイヤ
  - 写真ハッシュの Polygon アンカリングによる改ざん検知

■ プロダクト
  - 4 ポータル (Admin / Agent / Insurer / Customer) + モバイルアプリ
  - 130+ 本のマイグレーション、400+ の API、2,000+ のテストケース
  - 主要連携: Stripe / Square / Google Calendar / LINE / CloudSign / Polygon

■ 料金
```
