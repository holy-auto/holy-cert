## 2. サービス説明 (本文・約520字 / フォームの自由記述欄想定)

```
Ledra は、自動車のコーティング・PPF・板金・整備など「車に施工した事実」を
デジタル証明書として発行・流通させるマルチテナント SaaS です。

施工店の現場では、施工記録・顧客情報・車両情報・保険会社向けの確認情報が
分断されており、真面目に施工している事業者ほど書類対応や確認工数に時間を
取られています。一方で保険会社や中古車流通の現場では「いつ・誰が・どんな
施工をしたか」を後から検証する手段が乏しく、不正請求や品質トラブルの温床に
なっています。

Ledra は、施工店向けの業務 SaaS (予約 → 来店 → 作業 → 証明書発行 → 請求 →
入金を 1 画面に統合した案件ワークスペース) を入口に、発行された施工証明書を
顧客・保険会社・中古車市場が共通参照できる "車の施工台帳" に育てています。
証明書は QR / URL で公開検証でき、写真ハッシュは Polygon にアンカリング
することで第三者改ざん検知を実現しています。

施工店 (Admin) / 代理店 (Agent) / 保険会社 (Insurer) / 顧客 (Customer) の
4 ポータル + モバイルアプリ (iOS / Android) を提供し、業界に分散していた
価値の流れを 1 つのプラットフォームに集約することを目指しています。
```

## 4. ★ スキルパートナーに何を期待するか (CSP 向け資料の核)

CSP の差別化は資金より「スキルパートナー帯同」。よって**ここを具体・正直に
書けるかが当落を分ける**。`docs/ledra-goals-strategy-2026-05.md` で自社が
「最大のボトルネックは BizDev」と結論づけている事実とそのまま整合させる。

### 4-1. 最優先で伴走してほしい領域 (Ledra の真のボトルネック)

```
[A] エンタープライズ BizDev (最優先・最重要)
    - 損害保険会社との折衝・契約クロージング (Lighthouse: 損保ジャパン)
    - 大手 OEM 連携 (Woven by Toyota Partner Program 等) の提案・交渉設計
    - "B2B 埋込型は成約まで6〜12ヶ月" の長期商談を設計・推進できる人
    期待する人物像: 保険/モビリティ/大企業アライアンス経験のある事業開発者

[B] 店舗向け GTM / マーケティング
    - 数千〜1万社規模の施工店セグメントへの獲得チャネル設計
    - 代理店 (Agent) ネットワーク経由のパートナー獲得モデルの磨き込み
    - LP / オンボーディングの CVR 改善 (フリー→有料転換率)
    期待する人物像: 中小事業者向け SaaS の GTM/グロース実務経験者
```

> CSP の看板スキルパートナーは「マーケ/デザイン」が手厚い。[B] は CSP の
> 強みと正面から噛み合う。[A] は Ledra にとって最重要だが CSP の標準提供
> 領域とずれる可能性があるため、**「[B] で確実な価値、[A] は紹介・ネット
> ワーク提供でも可」** という温度差を送信文でも正直に伝えると刺さりやすい。

### 4-2. あると嬉しい (二次プライオリティ)

```
[C] デザイン: 4ポータルの IA 整理 / 保険会社向け管理画面の UX 改善
[D] ピッチ/IR: シード〜プレA のストーリーライン磨き込み・投資家紹介
[E] 採用: BizDev / CS の初期コアメンバー採用設計
```

### 4-3. 我々が提供できること (パートナー側の "やりがい" 訴求)

CSP は「志を支援したいスキルパートナーとの出会い」を謳う。片務的な"くれくれ"
ではなく、相手の関与価値も明示する。

```
- 自動車施工という一次情報を持つ Founder (detailing_holy 出身) と直接議論できる
- すでに本番稼働するプロダクト = 仮説検証ではなく "伸ばす" フェーズに関われる
- 保険 × ブロックチェーン × 業界ネットワークという、再現性の低い題材
- 関与いただいた成果が ARR・契約保険会社数という明確な指標で跳ね返る
```
