> 作成日: 2026年4月26日

## エグゼクティブサマリー

Ledra は自動車アフターマーケット（コーティング・PPF・鈑金塗装・ラッピング）向けの施工証明書 SaaS です。C2PA 署名・SHA-256 ハッシュ・Polygon ブロックチェーンアンカリングによる **改ざん不可能な施工記録** と、VIN 横断の **車両デジタルパスポート**（`/v/{vin}`）を提供しています。

Toyota Blockchain Lab が描く「車両ライフサイクル全体のブロックチェーン記録」において、**アフターマーケット施工の記録領域は現在空白地帯** です。Ledra は既に稼働している施工証明インフラとして、この空白を即座に埋められるポジションにあります。

本提案では、**3つのエントリーポイント** を提示します：

| # | 連携先 | 提案内容 | 期待インパクト |
|---|--------|---------|--------------|
| 1 | **Toyota Blockchain Lab** | 施工記録アンカーの TBL エコシステム統合 | 車両ライフサイクルデータの完全性向上 |
| 2 | **KINTO / Woven City** | Woven City 入居者向け施工証明・車両パスポートパイロット | PoC としての市場検証 |
| 3 | **トヨタ販売店ネットワーク** | ディーラー整備記録 + アフター施工の一元化基盤 | 中古車価値の定量的裏付け |

## 1. Ledra の現在地

### 1.1 プロダクト概要

Ledra は **施工証明書のデジタルインフラ** です。施工店がコーティング・PPF・鈑金塗装を完了した瞬間に、改ざん検知可能な証明書が生成されます。

```
施工完了
  │
  ├─ 施工写真 → C2PA 署名（コンテンツ真正性）
  │                 └─ SHA-256 ハッシュ
  │                        └─ Polygon PoS アンカリング（改ざん検知）
  │
  └─ 証明書 PDF → QR コード付き公開 URL（/c/{public_id}）
                        └─ VIN が紐づく場合 → 車両パスポート（/v/{vin}）
```

### 1.2 技術基盤（ブロックチェーン関連）

| 技術 | 実装状況 | 詳細 |
|------|---------|------|
| **C2PA 署名** | 本番稼働中 | コンテンツ真正性のマニフェスト署名（Phase 3b 実装済み） |
| **Polygon アンカリング** | 本番投入中 | 施工写真ハッシュを Polygon PoS にオンチェーン記録 |
| **車両デジタルパスポート** | 設計完了・実装着手 | VIN 単位で施工店横断の証明書を集約、`/v/{vin}` で公開 |
| **NFC タグ連携** | 本番稼働中 | 車両貼付タグ → パスポート URL に誘導 |
| **真正性グレード** | 本番稼働中 | `unverified` → `basic` → `verified` → `premium` の 4 段階 |

### 1.3 エコシステム

- **施工店（テナント）**: 複数店舗が Ledra 上で証明書を発行
- **代理店**: パートナー紹介ネットワーク
- **保険会社**: 損保会社が証明書を照会・案件管理するポータル
- **エンドユーザー**: QR / NFC で自分の施工証明書にアクセス

## 2. トヨタとの共鳴ポイント

### 2.1 Toyota Blockchain Lab (TBL) との整合性

TBL が掲げる「車両の製造〜廃棄の全履歴をブロックチェーンに記録」の構想において、以下のギャップが存在します：

```
製造・出荷          ← TBL がカバー
  ↓
ディーラー整備       ← 一部カバー（TBLOCK SIGN）
  ↓
アフターマーケット施工  ← ★ 空白地帯 ★（Ledra がカバー）
  ↓
中古車売買          ← 一部カバー（MON / Avalanche）
  ↓
廃車              ← TBL がカバー
```

Ledra はアフターマーケット施工の記録を **既に Polygon 上にアンカー** しており、TBL エコシステムへの統合は技術的に実現可能です。

**具体的な統合方法（案）：**
- Ledra の施工証明 Tx ハッシュを、TBL の車両履歴データベースに登録する API を開発
- MOBI DID（車両分散型識別子）との互換性を確保し、Ledra 証明書を DID ドキュメントの一部として参照可能にする
- 将来的には TBL の Ethereum ERC-4337 MOA（Mobility Oriented Account）と Ledra の車両パスポートを接続

### 2.2 KINTO × 安全運転証明との類似性

KINTO の SBT（Soulbound Token）安全運転証明（2024年実証実験）は、**行動の証明をブロックチェーン上で発行** するモデルです。Ledra の施工証明書はこれと同じ設計思想を持ちます：

| | KINTO SBT 安全運転証明 | Ledra 施工証明書 |
|---|----------------------|----------------|
| 対象 | ドライバーの安全運転行動 | 車両へのアフター施工 |
| 証明の主体 | KINTO | 認定施工店 |
| 改ざん耐性 | ブロックチェーン | C2PA + Polygon |
| 受益者 | ドライバー | 車両オーナー |
| 経済効果 | 保険料割引（想定） | 中古車査定価値向上 |

**提案**: KINTO サービス加入者のコーティング・PPF 施工に対して、Ledra 証明書を KINTO エコシステム内の「車両ケア証明」として統合。施工実績が KINTO スコアに反映される仕組みを設計する。

### 2.3 Woven City でのパイロット機会

Woven City（2025年9月開業）はブロックチェーンによるモビリティサービスの実験場です。入居者・訪問者の車両に対して：

- **施工証明のリアルタイム発行**: Woven City 内の整備・コーティング施設が Ledra を使用し、施工証明を即時発行
- **車両パスポートの実環境テスト**: `/v/{vin}` ページを Woven City の車両管理システムと連携
- **NFC タグの実地実証**: 車両に NFC タグを貼付し、スキャンで即座に施工履歴を確認
- **ブロックチェーン証明の生活統合**: P2P エネルギー取引（電力）と同じインフラで施工証明を管理

Woven City は閉じた環境での PoC に最適であり、Ledra の技術実証と TBL のデータ蓄積を同時に実現できます。

### 2.4 MON（Mobility Orchestration Network）との連携可能性

MON は Avalanche 上で車両をデジタル資産として扱うネットワークです（2025年8月、1,080万ドル投資）。

**連携シナリオ**:
- MON 上の車両デジタル資産に、Ledra の施工証明ハッシュをメタデータとして付加
- 施工実績が多い車両（PPF 施工済み・定期コーティング記録あり）はデジタル資産としての価値が上昇
- 中古車売買時に「Ledra 証明付き施工履歴」が MON 上で可視化される

## 3. 具体的な連携提案

### 提案 A: TBL との技術統合パートナーシップ

**内容**: Ledra の施工証明を Toyota Blockchain Lab の車両履歴エコシステムに組み込む。

**実施ステップ**:

1. **Phase 1（3ヶ月）**: 技術要件の共同定義
   - TBL のデータフォーマット（MOBI DID 規格等）の調査・互換性確認
   - Ledra の Polygon アンカー Tx を TBL システムに登録する API 仕様策定
   - セキュリティ・プライバシー要件の合意

2. **Phase 2（3ヶ月）**: PoC 実装
   - 参加施工店 5〜10 店でパイロット実施
   - 施工証明ハッシュを TBL システムへ自動送信
   - 車両オーナーが TBL ポータルで Ledra 証明書を参照できる UI を開発

3. **Phase 3（6ヶ月）**: 本番展開
   - Ledra 全テナントへのロールアウト
   - MOBI DID との完全互換性確保
   - 施工証明書の「TBL 認定マーク」付与による施工店の差別化支援

**期待される価値**:
- トヨタ側: 車両ライフサイクルデータの空白（アフターマーケット）を埋め、MON・中古車査定の精度が向上
- Ledra 側: TBL エコシステムへの参加により、施工店への「ブロックチェーン認定」訴求力が飛躍的に向上

---

### 提案 B: KINTO 連携 — 施工証明でスコアリングを強化

**内容**: KINTO サービス加入者が認定施工店で PPF・コーティングを施工した場合、Ledra 証明書を KINTO スコアに反映する。

**ユースケース**:
```
KINTO 加入者がコーティング施工
  │
  └─ 施工店が Ledra で証明書発行
          │
          └─ KINTO API Webhook で証明書データを通知
                  │
                  └─ KINTO スコアに「車両ケア実績」として反映
                          │
                          └─ 保険料割引・特典ポイント付与（将来）
```

**技術的実現性**: Ledra は既に Webhook 送信インフラ（LINE・CloudSign・Stripe 連携実績あり）を持つため、KINTO API への Webhook 追加は低コストで実現可能。

---

### 提案 C: Woven City パイロット

**内容**: Woven City の車両・施設向けに Ledra を採用し、施工証明の実環境実証を行う。

**スコープ**:
- Woven City 内のモビリティサービス拠点（整備・洗車・コーティング施設）に Ledra を導入
- 入居者・訪問者の車両 VIN を Ledra に登録し、施工のたびに車両パスポートが更新される体験を提供
- 車両の NFC タグをスキャンすると施工履歴が即表示されるデモを実施

**期待効果**:
- Woven City でのブロックチェーン活用ユースケースの拡充
- 「施工記録の透明性」が生活圏でどう機能するかの実データ取得
- Ledra にとって世界水準の PoC 実績の獲得

---

### 提案 D: ディーラー向け展開 — 整備記録 × 施工記録の統合

**内容**: トヨタ販売店（ディーラー）の整備記録と、同店舗内のコーティング・PPF 施工記録を Ledra 上で一元管理。

**現状の課題**:
- ディーラーでの定期整備は車検証・保証書で管理されるが、オプションの施工（コーティング・PPF等）は紙の施工証明書か口頭説明にとどまる
- 下取り査定時に施工実績が評価に反映されない

**Ledra の提供価値**:
- QR コード付き施工証明書をその場で発行 → 顧客にデジタル資産として提供
- 下取り・中古車販売時に Ledra 証明書を査定システムに連携（将来的にトヨタの中古車ビジネスと接続）

## 5. Ledra の競合優位性

### なぜ今 Ledra か

| 比較軸 | Ledra | 競合想定 |
|--------|-------|---------|
| 施工証明の真正性 | C2PA + Polygon の二重証明（業界初） | 写真管理のみ or ブロックチェーンなし |
| マルチテナント対応 | 複数施工店が同一インフラを共有 | 単一店舗向けシステムが多い |
| 保険会社連携 | 専用ポータル + 案件管理 | 連携なし |
| 車両パスポート | VIN 横断・テナント横断で集約 | 単一テナント内のみ |
| 既存稼働実績 | 本番稼働中・Polygon アンカー実績あり | コンセプト・PoC 段階 |
| 日本市場特化 | 日本語・日本市場向け設計 | グローバル標準のみ |

## 7. FAQと懸念事項の先読み

**Q. Ledra はスタートアップであり規模が小さい。大企業との連携に耐えられるか？**
A. Polygon アンカリング・Stripe 決済・Supabase など、エンタープライズグレードのインフラを採用。SLA・セキュリティ監査（`docs/AUDIT_REPORT_20260329.md` 参照）は既に整備済み。Vercel Enterprise プランへの移行も即対応可能。

**Q. データのプライバシーはどう担保するか？**
A. ブロックチェーンに記録するのは施工写真の **SHA-256 ハッシュ**のみ（個人情報・画像データ自体はオフチェーン）。顧客 PII は表示時にマスク処理（`src/lib/certificate/publicData.ts`）。GDPR 類似のオプトアウト機構も実装済み。

**Q. トヨタ独自のブロックチェーン（Avalanche/Ethereum）と Polygon の違いは問題か？**
A. 施工証明ハッシュはチェーン非依存のデータです。Polygon への記録に加え、必要であれば Avalanche（MON）や Ethereum（TBL テックペーパーの ERC-4337）へのクロスチェーン記録にも対応可能。技術的な障壁は小さい。

**Q. 既存のプレイヤー（TBLOCK SIGN 等）との棲み分けは？**
A. TBLOCK SIGN はサプライチェーン向け電子契約（R3 Corda ベース）であり、アフターマーケット施工証明とは対象が異なる。競合ではなく補完関係。
