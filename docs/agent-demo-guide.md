## 1. 本書の使い方

- **商談前**: 第 2 章「30 秒の全体像」と、相手のロール（施工店 / 代理店 / 保険会社）に合わせた第 5 章のシナリオを 1 つ流し読みしてください。
- **商談中**: PC を画面共有しながら、第 5 章の **デモシナリオ** を順番に実演してください。各シナリオは「ストーリー → 操作手順 → トーク例 → 訴求ポイント」の 4 点セットです。
- **商談後**: 質問に対しては第 7 章（仕様サマリー）と第 8 章（FAQ）から回答してください。即答できない場合は「持ち帰り → Ledra 運営に確認」が原則です。

## 2. Ledra 全体像（30 秒で説明する）

> 「Ledra は、自動車のコーティング・フィルム・ラッピングなど **施工の事実をデジタル証明書として残し、検証できるようにする SaaS** です。
> 施工店は紙の証明書から脱却して 業務効率化＋顧客満足度向上 が実現でき、保険会社は **施工の真正性を即時に確認** できるため保険金査定が早くなります。
> 4 つのポータル（施工店 / 代理店 / 保険会社 / 顧客）で構成されており、それぞれの役割に最適化された画面を提供します。」

### 4 つのポータル

| ポータル | 対象 | 主な役割 | URL プレフィックス |
|---|---|---|---|
| **Admin** | 施工店スタッフ | 証明書発行、車両・顧客管理、予約、請求、BtoB | `/admin/*` |
| **Agent** | 代理店パートナー | 紹介管理、コミッション、レポート | `/agent/*` |
| **Insurer** | 保険会社査定担当 | 証明書クロステナント検索、案件管理、SLA | `/insurer/*` |
| **Customer** | 一般エンドユーザー | 自分の証明書をスマホで閲覧 | `/c/*`, `/customer/[tenant]/*` |

## 4. デモ環境の前提・お願い

商談相手にも一言伝えるとスムーズです。

- **データはいつでもリセットされる可能性があります**（夜間ジョブで再シードする運用想定）。重要な書き込みは行わないでください。
- **メール送信は本物の宛先に届きません**。Resend のサンドボックスドメイン経由なので、デモ用に画面上の操作のみ確認してください。
- **決済は Stripe テストモード**で動作します。テスト用カード番号 `4242 4242 4242 4242`（任意の将来日 / 任意の CVC）で課金フローまで体験できます。実カードは絶対に使わないでください。
- **デモ環境とお客様の本番環境は完全に分離**されています。デモで作った操作ログ・証明書がそのまま本番に出ることはありません。

## 5. ポータル別 デモシナリオ

各シナリオは **5〜10 分** を想定。商談時間に合わせて 1 つに絞っても、複数組み合わせても OK です。

### 5.1 施工店ポータル（Admin）

#### シナリオ A. 「飛び込み案件 → 証明書発行 → 顧客に QR 共有」（10 分・最頻出）

> **ストーリー**: 予約なしで来店された一見のお客様に、その場でガラスコーティング施工 → 証明書を発行 → QR で顧客のスマホに渡すまでを見せます。「現場で完結する」を伝えるシナリオです。

1. `/login` から `demo@ledra-motors.example` でログイン
2. ダッシュボード（`/admin`）で **KPI カード**（証明書数 / 顧客数 / 未回収額）を 5 秒解説
3. クイックアクションの **「🏃 飛び込み案件」** をクリック → `/admin/jobs/new`
4. タイトルに「ガラスコーティング 飛び込み」と入力 → ステータス「🚪 来店・受付」を選択 → 作成
5. 案件ワークフロー画面（`/admin/jobs/[id]`）が開く
   - **ステータスステッパー** をクリックして `arrived → in_progress → completed` まで進める
6. 「次アクション」パネルから **🪪 証明書を発行** → `/admin/certificates/new`（車両 ID・顧客 ID は URL で自動引き継ぎ）
7. 施工内容を選んで写真をアップロード → **発行**
8. 完了画面で **QR コード** を表示 → スマホで読み取って `/c/[public_id]` を開いて見せる

**トーク例**
- 「予約なしのお客様も、タイトル 1 つ書けば数秒でワークフローに乗ります」
- 「証明書は QR コードでお客様のスマホに渡るので、紙の保管・紛失リスクがゼロになります」
- 「同じ画面の中で『来店 → 作業 → 証明書 → 請求』まで進められるので、複数画面を行き来しません」

**訴求ポイント**
- ✨ **時短**: 予約・車両・顧客 ID を毎画面で再入力しない
- ✨ **重複検知**: すでに有効な証明書があるとボタンに `(発行済)` 表示で過剰発行を防ぐ
- ✨ **属人化排除**: スタッフ全員が同じステップで業務を進められる

---

#### シナリオ B. 「車両のサービス履歴を 1 画面で振り返る」（5 分）

> **ストーリー**: 既存顧客の車両ページを開くだけで、過去の施工・予約・NFC 書き込みが時系列で全部見える。リピート提案・損保問合せ対応の高速化を訴求します。

1. `/admin/vehicles` → 「LEXUS RX 500h F SPORT」をクリック
2. **サービス履歴タイムライン** を見せる
   - 証明書発行・予約・NFC 書込が**色分けバッジ**で識別
   - 各イベントをクリックすると証明書詳細・案件ワークフローに即遷移
3. 「保険会社から『この車両、過去にどんな施工をしたか教えて』と聞かれた時、この 1 画面で即答できます」と説明

**訴求ポイント**
- ✨ **問い合わせ対応の即応性**
- ✨ **顧客への次回提案ネタが見える化**

---

#### シナリオ C. 「顧客 360° ビュー」（5 分）

1. `/admin/customers` → 「山田 太郎」をクリック
2. 4 タブ（**車両 / 証明書 / 予約・案件 / 請求**）を切り替えて「1 顧客の全コンテキスト」を見せる
3. タブ右上の **「🪪 証明書発行」「🏃 飛び込み案件」「💰 請求書作成」** ボタンから次アクションへ進める

#### シナリオ D. 「BtoB マーケットプレイス」（5 分・上級店向け）

1. `/admin/btob` を開く → 自店の得意分野・在庫を掲載できることを説明
2. `/admin/orders` → 受注・発注の一覧、ステータス管理（申請中 → 承認 → 作業中 → 完了）
3. `/admin/market-vehicles` → 中古車を相互掲載できる

#### シナリオ E. 「経営分析・パートナーランク」（5 分・経営者向け）

1. `/admin/management` で売上推移・顧客単価・リピート率を表示
2. ダッシュボード上部の **パートナーランク**（プラチナ / ゴールド / シルバー / ブロンズ / スターター）を解説
3. 「実績がそのままランクに反映され、Ledra 内での露出・優遇が変わります」

### 5.2 代理店ポータル（Agent）

> **ストーリー**: 代理店ご自身の業務イメージを掴んでいただくために、ご自身がログインして実際に触っていただくのが一番です。

1. `/agent/login` から `demo@agent.ledra.test` でログイン
2. **ダッシュボード**（`/agent`）の KPI カードを確認
   - 紹介総数 / 契約成立数 / 成約率 / 今月コミッション / 累計コミッション
   - 月別コミッション推移チャート
3. **新規紹介**（`/agent/referrals/new`）→ 紹介フォームを 1 件入力してみる
4. **紹介リンク**（`/agent/referral-links`）→ トラッキング URL の生成
5. **コミッション**（`/agent/commissions`）→ 支払履歴・明細
6. **営業資料**（`/agent/materials`）→ 8 件の PDF（サービス紹介 / 料金プラン / 契約書テンプレート ほか）からダウンロード
7. **契約書**（`/agent/contracts`）→ 署名完了 / 送信済 / 閲覧済 の 3 状態を見せる
8. **ランキング**（`/agent/rankings`）→ 代理店間の実績比較
9. **キャンペーン / 研修**（`/agent/campaigns`, `/agent/training`）→ 学習・特典コンテンツ

**トーク例**

### 5.3 保険会社ポータル（Insurer）

> **ストーリー**: 損保査定担当が「事故車の施工証明書を即座に確認したい」というユースケース。クロステナント検索（全施工店を横断）が最大の差別化ポイントです。

1. `/insurer/login` から `demo@insurer.ledra.test` でログイン（admin ロール）
2. **ダッシュボード**（`/insurer`）→ 案件サマリー（未対応 / 対応中 / 今日更新）を確認
3. **証明書検索**（`/insurer/search`）
   - 「LEDRA-DEMO-0001」を検索 → 結果テーブル → 詳細遷移
   - 顧客名「山田」、車両「クラウン」など複数キーで検索体験
4. **保存済み検索**（左メニュー or `/insurer/search?saved=...`）
   - 「直近 30 日 アクティブ証明書」「トヨタ車 検索」「無効化された証明書」の 3 件をワンクリックで再検索
5. **証明書詳細**（`/insurer/c/LEDRA-DEMO-0001`）
   - 施工内容・写真・施工店情報・真正性の表示
6. **ウォッチリスト**（`/insurer/watchlist`）→ 監視対象の証明書登録
7. **案件管理**（`/insurer/cases`）
   - 一括ステータス変更、テンプレート（`/insurer/templates`）、自動振り分けルール（`/insurer/rules`）、SLA（`/insurer/sla`）
8. **CSV / PDF エクスポート**（Pro プラン機能）
9. **ロール切替デモ**: 一度ログアウトして `viewer@insurer.ledra.test` で入り直し → 「メンバー管理」「設定変更」が表示されないことを確認
10. **監査ログ**（`/insurer/audit`）→ `auditor@insurer.ledra.test` でも閲覧可能

**トーク例**
- 「Ledra に登録された **全施工店の証明書を 1 つの検索画面から横断的に** 探せます。施工店ごとに窓口を変える必要がありません」
- 「保存済み検索とウォッチリストで、定型の照会業務を圧倒的に短縮できます」
- 「ロールベースで 監査担当には書き込み権限を渡さない などの内部統制も可能です」

**訴求ポイント**
- ✨ **クロステナント検索**（保険会社にしか提供しない機能）
- ✨ **SLA・自動振り分けで案件管理を組織化**
- ✨ **3 ロール（admin / viewer / auditor）で内部統制**

## 8. 想定質問（FAQ）

### Q1. データは私たちの店舗だけのものになりますか？他店から見えませんか？
**A.** はい。Supabase の RLS（Row Level Security）に加え、アプリケーション層でも `tenantId` で必ず絞り込む二重防御を行っています。**保険会社ポータルだけ** がクロステナント検索を行えますが、これは施工証明書の「公開メタデータ」（公開 ID、車両情報、施工概要）に限定しており、施工店の経営情報や顧客連絡先などは閲覧できません。

### Q2. 証明書が「本物」であることを保証する仕組みは？
**A.** 3 段階で担保しています。
1. Ledra のサーバ署名 + 公開 URL（`/c/[public_id]`）で誰でも検証可能
2. （任意）施工写真の SHA-256 ハッシュを **Polygon ブロックチェーン** にアンカリング
3. （オプション）C2PA 準拠の写真メタデータ署名

### Q3. インターネット回線が落ちたら現場で施工証明書は発行できないのでは？
**A.** 現状はオンライン前提です。回線が不安定な現場ではモバイル API + ローカルキャッシュ運用を推奨しています。完全オフライン対応はモバイルアプリのロードマップに含まれています。

### Q4. 既存の業務システム（販売管理など）と連携できますか？
**A.** Square OAuth、Google Calendar、CloudSign は標準連携。それ以外は CSV インポート（車両一覧）/ CSV・PDF エクスポート（保険会社）と Webhook 受信で対応します。専用 API 連携は Enterprise プランで個別対応します。

### Q5. 解約時にデータは持ち出せますか？
**A.** 顧客 / 車両 / 証明書 / 請求書はすべて CSV / PDF でエクスポート可能です。Enterprise プランでは生 SQL ダンプの提供も検討可能です。

### Q6. デモアカウントの中身を私たちが書き換えていいですか？
**A.** **問題ありません**。デモ環境は定期的にリセットされる前提です。むしろ商談相手の前で、実際に証明書を発行して QR を読み取ってもらうところまで体験させると刺さります。

### Q7. 顧客にメールが本当に届きますか？
**A.** **デモ環境では実宛先には届きません**。サンドボックスです。本番環境では Resend 経由で確実に配信されます（配信ステータスも Webhook で取得）。

### Q8. 施工証明書のテンプレート（デザイン）は変えられますか？
**A.** はい。`/admin/template-options` から店舗ロゴ・カラー・レイアウトを選べます。専用テンプレ製作は Ledra 運営に発注いただけます（`/admin/template-options/order`）。

### Q9. 月額の最低コミットや初期費用はありますか？
**A.** 最新の料金条件は `/agent/materials` の **Ledra 料金プラン一覧** をご確認ください。代理店ご担当者は Ledra 運営に直接ご確認いただいても結構です。

### Q10. 代理店として申請後、どれくらいで承認されますか？
**A.** 通常 3〜5 営業日です。`/agent/apply` から申請後、`/agent/apply/status` で進捗を確認できます。

## 9. トラブル時の対処

### デモアカウントでログインできない

1. メールアドレス・パスワードを再確認（`Demo1234!` の `!` は半角）
2. ログイン URL が正しいかを確認（施工店 = `/login`、代理店 = `/agent/login`、保険会社 = `/insurer/login`）
3. それでもダメなら **Ledra 運営にご連絡** ください（夜間ジョブでアカウントが再作成された直後は数分待ってから再試行）

### デモデータが空 / 期待通り表示されない

- デモシードがリセット直後の可能性があります
- 運営側で `npx tsx scripts/setup-demo-tenant.ts` `setup-demo-tenant-user.ts` `setup-demo-agent.ts` `setup-demo-insurer.ts` を再実行することで復旧します

### 商談中に画面が固まった / 500 エラー

- リロード → ダメなら別タブで再ログイン
- それでもダメな場合は Sentry にエラーが上がっているはずなので、**お時間を改めてご案内** とお詫びし、運営に Slack で連絡してください

### お客様から仕様の細部について踏み込んだ質問が来た

- 即答できないことは「持ち帰り → Ledra 運営に確認 → 翌営業日までに回答」が公式対応です
- ハルシネーションを避けるため、推測でお答えしないでください
