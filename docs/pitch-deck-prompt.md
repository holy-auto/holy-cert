## 使い方

1. 下の `text` コードブロックを丸ごとコピーし、Claude / GPT 等に貼る。
2. 出力された Marp Markdown を `deck.md` として保存。
3. 変換: `npx @marp-team/marp-cli deck.md --pdf`（または `--pptx`）。
4. **送付前に `【要記入：…】` を実数で埋める**（創業年・チーム・トラクション実数・調達額/ステージ等）。憶測値を入れない。

## EN 版が要るとき

## 設計の要点（なぜこの形か）

- **誇張禁止**: 保険会社とは未接触（pre-contact）。SOMPO 等は「標的／紹介を求めたい先」としてのみ記述。
- **Insurtech 主軸**: モビリティは将来1行。`/insurer/*` が実装済みで「触れる」唯一のトラック。
- **投資家受けの資金使途**: 科目でなく到達マイルストーンを見出しに。「build raise でなく資本効率の高い GTM/転換 raise」。人件費は機能バケツに内包（"Key Hires" 独立バケツは作らない）。

# 【制約】（最重要・違反厳禁）
1. 保険会社とのトラクション・関係性を一切誇張しない。**損保ジャパン含む保険会社とは現時点で未接触（pre-contact）**。「導入済み」「協業中」「PoC実施中」等は書かない。SOMPO等は「経営が選んだ標的／紹介を求めたい先」としてのみ記述。
2. ポジションは Insurtech。モビリティ（トヨタ Woven City、車両パスポート `/v/[vin]`）は「将来の拡張」としてロードマップに1行触れる程度。前面に出さない。
3. 実数が必要な箇所（創業年、メンバー数・経歴、有料テナント数、MRR/ARR実績、累計証明書発行数・オンチェーン件数、調達ステータス・金額）は決して創作せず、必ず `【要記入：◯◯】` というプレースホルダにする。
4. トラクション・スライドは正直に。「プロダクトはライブ、店舗側は初期トラクション（実数=要記入）、保険側は未接触（接点獲得が本プログラム参加の目的）」という構成にする。
5. 数字・比較は【事実シード】の範囲のみ。出典が言える範囲で控えめに。
6. 絵文字は使わない。1スライドの箇条書きは最大6行、各行およそ全角25字以内。冗長な文章にしない。

# 【資金使途スライド仕様（投資家向け・最重要）】
- 見出しは金額でなく到達状態を言い切る（リーン・シードで現実的な範囲に縮小）:「プロダクト完成済み・有料テナント稼働・保険接点ゼロ → 〔保険1社を 紹介→PoC→契約 まで〕＋〔SOC2/ISO27001 “準備” 完了（フル認証でない）〕、約12ヶ月runway確保（ファウンダー主導GTM前提）。指定工場ネットワーク本格展開と保険1社の有償本番（スケール）はシリーズAマイルストーンへ繰り延べ」
- 投資ストーリーの核を1行で明示:「これは"開発のための調達"ではなく、既存プロダクトを初回保険成約と本番化に転換する"資本効率の高いGTM/転換ラウンド"」
- 総額は **シード ¥3,000〜5,000万（想定時期【要記入】）** を明示し、「リーン・シード（意図的に狭いスコープ）」と1行添える。配分は成果（機能）バケツ4つ以内・各バケツは人件費込み（"Key Hires"の独立バケツは作らない）。比率は%レンジで併記:
  | 区分（人件費込み） | %レンジ | 何を買うか |
  |---|---|---|
  | GTM：保険1社の紹介→PoC→契約・限定パイロット | 35–45% | GTM 1名（エンプラ営業/CS兼務）＋ファウンダー主導。指定工場は本格網でなく限定パイロット規模 |
  | エンプラ調達・セキュリティ/コンプラ | 15–20% | SOC2/ISO27001「準備」・ペネトレ・DPA/法務・SSO（フル認証はシリーズA） |
  | プロダクト深掘り・本番化（新規開発でなく） | 20–25% | /v/[vin]、Polygon Mainnet本番化、1社目PoC連携の最小実装（2社目テンプレ化はシリーズA） |
  | 運転資金・予備 | 10–15% | 約12ヶ月リーンrunway、1社目PoC期間の運用コスト、予備 |
- レンジ合計が100%に収まるよう最終調整。図版案コメントで「横棒1本 or ドーナツ1点（4区分）」を指示。
- ユニットエコノミクスは slide 9 の正準値を1行で再掲（数値を二重定義しない）。
- 誇張禁止と整合:「導入拡大」でなく「保険1社を獲りに行く」配分として記述。保険は未接触のまま。
