# Tap to Pay on iPhone - Distribution承認チェックリスト

Apple Distribution Entitlement 取得のための要件と Ledra 現状の対応マッピング。

**ステータス凡例:**
- ✅ 実装済み
- ⚠️ 部分実装（要追加対応）
- ❌ 未実装（新規実装必須）
- N/A 該当なし

## 推奨アプローチ

### Path A: 暫定リリース → 後日 TTP 対応 (推奨)
1. 今のビルド (QR/現金/振込) で TestFlight → App Store 提出
2. 並行して Distribution 承認用に上記実装
3. 動画撮影 → Apple 提出 → 1〜4週間後に承認
4. TTP 入りバージョンを段階リリース (Phased Release)

### Path B: TTP 対応完了まで待つ
1. 1〜2週間かけて全要件実装
2. 動画撮影 → Apple 提出
3. 承認後にローンチ

---

## 例外: 配布チャネルを変更する選択肢

App Store 経由ではない配布なら、要件 2.x (アプリ内サインアップ) を回避できる：

- **Custom Apps**: 既知の組織にのみ配布。Apple Business Manager 経由
- **Unlisted Apps**: App Store には載せず URL 経由のみ
- **Apple Developer Enterprise Program (ADEP)**: 自社内配布のみ

Ledra が完全 B2B (一般消費者向け配布をしない) なら Custom Apps 配布が現実的。これなら 2.x 要件免除。
