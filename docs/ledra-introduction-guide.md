# Ledra 導入ガイド

> 自動車施工証明書 SaaS プラットフォーム

---

## 1. Ledra とは

Ledra は、自動車施工（コーティング・フィルム・ラッピング等）の記録を **デジタル証明書** として発行・管理するマルチテナント SaaS プラットフォームです。

### 解決する課題

| 従来の課題 | Ledra による解決 |
|---|---|
| 紙の施工証明書は紛失・劣化しやすい | QRコード付きデジタル証明書で永続的に管理 |
| 証明書の真正性を第三者が検証できない | 公開URLで誰でも即座に検証可能 |
| 施工店の業務が分散し非効率 | 予約→作業→証明書→請求を1画面で完結 |
| 保険会社が施工実績を確認しづらい | 専用ポータルで証明書を即検索・照会 |
| 施工店間の連携手段がない | BtoBマーケットプレイスで受発注・商談 |

## 2. プラットフォーム構成

Ledra は **4つのポータル** で構成され、各ユーザーの役割に最適化された画面を提供します。

```
┌─────────────────────────────────────────────────────┐
│                    Ledra Platform                    │
├──────────┬──────────┬──────────┬─────────────────────┤
│  Admin   │  Agent   │ Insurer  │     Customer        │
│ 施工店   │ 代理店   │ 保険会社  │     顧客            │
├──────────┼──────────┼──────────┼─────────────────────┤
│証明書発行│紹介管理  │証明書照会│証明書閲覧           │
│車両管理  │コミッション│案件管理 │QRコードアクセス     │
│予約管理  │ランキング│分析     │PDF出力              │
│請求・会計│レポート  │SLA管理  │                     │
│BtoB受発注│研修     │監査ログ  │                     │
│経営分析  │サポート  │セキュリティ│                   │
└──────────┴──────────┴──────────┴─────────────────────┘
```

### 2.1 施工店ポータル（Admin）

施工店の日常業務を一元管理するメインポータルです。

**主要機能:**
- **証明書管理** — 新規発行、一覧、PDF出力、バッチ出力、無効化、複製
- **車両管理** — 車両登録、車検証OCR読取、CSVインポート
- **顧客管理** — 顧客情報、紐づき証明書、サマリー
- **予約管理** — 予約CRUD、ステータス管理、Google Calendar連携
- **案件ワークフロー** — 予約→来店→作業→証明書→請求を1画面で統合管理
- **請求・帳票** — 請求書作成、PDF出力、合算請求、ステータス管理
- **POS会計** — Stripe Terminal連携、レジセッション、日次レポート
- **Square売上連携** — OAuth接続、売上データ同期、注文管理
- **経営分析** — 売上推移、顧客単価、リピート率
- **BtoBプラットフォーム** — 施工店間マーケットプレイス、受発注、商談
- **NFC管理** — NFCタグ登録、車両紐付け、書き込み
- **設定** — 店舗設定、メンバー管理、ブランド証明書テンプレート、課金管理

### 2.2 代理店ポータル（Agent）

パートナー代理店が施工店を紹介し、コミッションを管理するポータルです。

**主要機能:**
- 紹介管理・トラッキングリンク生成
- コミッション履歴・明細
- ランキング・実績レポート
- キャンペーン・研修コンテンツ
- サポートチケット

### 2.3 保険会社ポータル（Insurer）

保険会社が施工証明書を検索・照会し、案件を管理するポータルです。

**主要機能:**
- 証明書検索（public_id / 顧客名 / 車両情報）
- 案件管理（一括操作、テンプレート、自動振り分け、SLA）
- ウォッチリスト
- 検索分析・案件レポート
- CSV/PDFエクスポート

### 2.4 顧客ポータル（Customer）

エンドユーザーが自分の証明書を閲覧するシンプルなビューです。

**主要機能:**
- QRコード / URLから証明書を直接閲覧
- 証明書の真正性検証
- PDF出力
- 顧客マイページ（テナント別ログイン）

## 3. 主要ワークフロー

### 3.1 施工証明書の発行（基本フロー）

```
顧客来店
  │
  ▼
予約チェックイン ─── 飛び込み案件の場合は /admin/jobs/new から即開始
  │
  ▼
車両登録（車検証OCR or 手動入力）
  │
  ▼
施工実施 → 写真撮影・アップロード
  │
  ▼
証明書作成（施工内容・写真・備考を入力）
  │
  ▼
証明書発行 → QRコード自動生成
  │
  ▼
顧客にQRコード / URL共有
  │
  ├──→ 顧客がスマホで証明書閲覧
  └──→ 保険会社が証明書を検索・照会可能に
```

### 3.2 案件ワークフロー（統合ワークスペース）

`/admin/jobs/[id]` で予約から請求までを1画面で管理できます。

```
  ┌─────────┐    ┌─────────┐    ┌──────────┐    ┌─────────┐
  │ 予約確定 │ →  │  来店   │  → │  作業中  │  → │  完了   │
  │confirmed│    │ arrived │    │in_progress│   │completed│
  └─────────┘    └─────────┘    └──────────┘    └─────────┘
                                                      │
                                     ┌────────────────┤
                                     ▼                ▼
                               証明書発行         請求書作成
                            （車両・顧客ID      （金額・明細
                              を自動引き継ぎ）     を自動反映）
```

**ポイント:**
- ステータスステッパーで1クリック進行
- 車両ID・顧客IDをURLパラメータで次画面に引き継ぎ、再入力不要
- 重複検知機能で証明書の過剰発行や二重請求を防止
- 飛び込み案件も `/admin/jobs/new` から数秒でワークフローに乗せられる

### 3.3 予約→会計フロー

```
予約登録 → Google Calendar同期 → チェックイン → 作業中
    → 作業完了 → POS会計 or 請求書発行 → Square/Stripe決済
    → 売上データ連携 → 経営分析に反映
```

## 4. 外部サービス連携

Ledra は多数の外部サービスと連携し、業務の自動化と効率化を実現します。

| サービス | 用途 |
|---|---|
| **Stripe** | サブスクリプション課金、Connect決済、Terminal POS |
| **Square** | 売上連携（型定義・移行スキーマ整備済み、決済フローはロードマップ） |
| **Google Calendar** | 予約スケジュール同期 |
| **LINE** | メッセージング連携 |
| **電子署名（内製）** | ECDSA P-256 ベースの自前実装（CloudSign 等の外部連携はロードマップ） |
| **Resend** | トランザクションメール（認証・通知） |
| **Supabase** | データベース、認証、リアルタイム通信、ファイルストレージ |
| **Upstash** | Redis キャッシュ、レートリミット、QStash非同期処理 |
| **Sentry** | エラー監視・パフォーマンス追跡 |
| **Vercel** | ホスティング、Analytics、Speed Insights |

| 権限カテゴリ | 例 |
|---|---|
| 閲覧系 | `dashboard:view`, `certificates:view`, `vehicles:view` |

## 7. セットアップ手順

### 前提条件

- Node.js 18+
- npm（パッケージマネージャ）
- Supabase プロジェクト
- Stripe アカウント（決済機能を使用する場合）

### クイックスタート

```bash
# 1. リポジトリをクローン
git clone <repository-url>
cd Ledra

# 2. 依存パッケージをインストール
npm install

# 3. 環境変数を設定
cp .env.example .env.local
# .env.local を編集して各サービスのキーを設定

# 4. 開発サーバーを起動
npm run dev
```

`http://localhost:3000` でアプリケーションにアクセスできます。

### 主要な環境変数

| 変数 | 説明 |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase プロジェクトURL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase 匿名キー |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase サービスロールキー |
| `STRIPE_SECRET_KEY` | Stripe シークレットキー |
| `RESEND_API_KEY` | Resend メール送信キー |

詳細は `.env.example` を参照してください。

### 開発コマンド

| コマンド | 説明 |
|---|---|
| `npm run dev` | 開発サーバー起動 |
| `npm run build` | プロダクションビルド |
| `npm run lint` | ESLint 実行 |
| `npm run test` | ユニットテスト実行（Vitest） |
| `npm run test:e2e` | E2Eテスト実行（Playwright） |
| `npm run format` | Prettier でコード整形 |

## 8. API 概要

Ledra は 200以上の API エンドポイントを提供しています。

| ポータル | API数 | 主な用途 |
|---|---|---|
| Admin | 100+ | 証明書CRUD、車両、顧客、予約、請求、BtoB |
| Agent | 15+ | 紹介管理、コミッション、レポート |
| Insurer | 25+ | 証明書検索、案件管理、分析 |
| Public | 30+ | 証明書公開ビュー、認証、Webhook |
| Mobile | 15+ | NFC、POS、作業進捗、チェックイン |
| Webhook | 4 | Stripe、LINE、Resend、社内署名イベント（CloudSign 受信はロードマップ） |
| Cron | 6 | 課金、フォローアップ、同期、監視 |

### モバイル API（施工現場向け）

施工現場でのモバイルデバイス利用を想定した専用 API を提供しています。

- 証明書のアクティベート / 無効化
- NFCタグの紐付け・書き込み
- POS決済処理
- 作業進捗のリアルタイム更新
- 来店チェックイン / 作業開始 / 作業完了

## 10. デザインシステム

Ledra は統一されたデザインシステムを採用しています。

- **マーケティングサイト**: ダークテーマ・演出的・CV重視
- **アプリケーション**: ライトテーマ（ダーク切替可能）・機能的・業務効率重視
- **UIコンポーネント**: Button, Badge, Card, Modal, DataTable, Toast 等を標準化
- **デザイントークン**: カラー、フォント、角丸、シャドウを CSS変数で一元管理

詳細は `DESIGN_SYSTEM.md` を参照してください。

## 付録: 用語集

| 用語 | 説明 |
|---|---|
| **テナント** | Ledra に登録された1つの施工店（組織単位） |
| **証明書（Certificate）** | 施工内容を記録したデジタル証明書。QRコード付き |
| **public_id** | 証明書の公開識別子。URL `/c/[public_id]` でアクセス |
| **案件（Job）** | 1つの予約に紐づく業務の流れ全体 |
| **パートナーランク** | 施工店の実績に応じたランク（プラチナ〜スターター） |
| **BtoB** | 施工店間の取引・受発注プラットフォーム |
| **コミッション** | 代理店が紹介成功時に受け取る報酬 |
| **POS** | Point of Sale（店頭決済） |
| **NFC** | Near Field Communication。車両に貼付するタグ |
| **C2PA** | Coalition for Content Provenance and Authenticity。写真の真正性保証 |
| **Polygon アンカリング** | ブロックチェーンへの施工写真 SHA-256 ハッシュ記録（改ざん防止） |
