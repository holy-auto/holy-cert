# 事例記事 制作ガイド

Ledra の導入事例を書くとき／取材するときの社内ガイドです。文体・構成・現場との
進め方を揃えることで、記事ごとの質のぶれを抑えます。

---

## 基本方針

### ブランドトーン

- **詩的すぎず、実務的すぎず**。SaaS っぽさを出すより、施工店の現場の体温を
  通す
- **数字で語れる部分は数字で**。「導入時間が短縮」ではなく「1件あたり15分 →
  3分」と書く
- **現場の言葉をそのまま載せる**。リライトで磨きすぎない
- **成功だけでなく、導入過程の工夫も書く**。失敗も含めて語ると信頼される
- 専門用語は避けない。ただし初出時に簡単に補足

### やらないこと

- ベンダーロック臭を出す（「Ledra さえあれば〜」）
- 他社比較を感情的にやる
- 数字を盛る／推測で書く
- 現場に確認していない個別エピソードを創作する

---

## 記事の構成（テンプレート準拠）

1. **お客様について** — 簡潔に（箇条書き中心）
2. **導入前の課題** — 3〜5個、具体的に
3. **Ledra 導入の流れ** — きっかけ・導入プロセス
4. **導入後の変化** — 定量 + 定性
5. **これからの展望** — 2〜3文
6. **Ledra からのひとこと** — 2〜3文

各セクションの推奨文字数の目安:

| セクション | 推奨字数 |
|---|---|
| お客様について | 100〜150 字 |
| 導入前の課題 | 300〜400 字 |
| 導入の流れ | 400〜600 字 |
| 導入後の変化 | 500〜700 字 |
| 展望 | 100〜150 字 |
| ひとこと | 100〜150 字 |
| **合計** | **1,500〜2,000 字** |

## 取材の進め方

### 0. 前提条件の確認

- 事例掲載の承諾書（CloudSign テンプレート `CASE_STUDY_CONSENT`）を締結済み
- 公開範囲の確認: ロゴ掲載 / 会社名掲載 / スタッフ実名 / 写真掲載

### 1. 取材前ヒアリングシート（30分）

- 導入の決め手は何だったか（上位 3 つ）
- 導入前、1日のどこに一番時間が取られていたか
- 導入初週にいちばん大変だったのは何か
- 現在、Ledra のどの機能を一番よく使っているか
- Ledra 導入後、お客様（エンドユーザー）の反応はどう変わったか
- 他の施工店に紹介するとき、どう説明しているか
- （数値）月間発行数、平均客単価、再来店率などで変化のあったもの

### 2. 現場撮影（60分 / 任意）

- 施工中のシーン（手元のアップ）
- 作業完了後の車両とタブレット
- スタッフの作業風景
- 店舗外観・看板
- 代表インタビューの正面カット

撮影ガイドは別紙 `docs/marketing/photo-guidelines.md` に。

### 3. ドラフト作成（Ledra 編集側）

- テンプレート `src/content/cases/_template.mdx` をコピーして `<slug>.mdx` に
- `draft: true` のままで PR を出し、内部レビューを先に通す
- お客様レビューに渡す前に、数字の桁・固有名詞・職名を重点チェック

### 4. お客様レビュー

- PR のプレビュー URL を先方にお送りし、以下の観点で確認いただく:
  - 数字の正確性
  - 発言内容の意図と表現
  - 公開して差し支えない固有名詞の範囲
- フィードバック反映 → 再確認 → 承認を待つ

### 5. 公開

- `draft: false` に変更
- OG 画像（動的生成）が意図通りに出ているかを Slack プレビューで確認
- `/cases` 一覧に反映されていることを確認
- ニュース枠に「【事例】◯◯社の導入事例を公開しました」を投下

## SEO のお作法

- `title`: 50〜60 文字以内、会社名＋業態＋成果を含める
- `excerpt`: 90〜120 文字、成果数字を 1 つ入れる
- `slug`: `<company-name>-case` または `<business-type>-<name>-case`
  全て kebab-case、日本語を入れない
- `tags`: 2〜4 個、業態と機能で区分

---

## よくある NG パターン

1. **抽象的すぎる効果**: 「業務効率が上がりました」→「書類作成時間が 1 日
   60 分短縮されました」
2. **Ledra の自慢だけ**: 現場側の主体性を消さない
3. **発言の過剰編集**: 現場の言葉を校正で薄めない
4. **導入課題の一般論化**: 「業界全体として〜」より「うちの店として〜」
5. **担当者名の言い換え**: 肩書きと名前は正確に（敬称も）
