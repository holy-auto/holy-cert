# 大企業交渉戦略 — トヨタとの力学を有利に動かす

> 作成日: 2026年4月26日
> 用途: 交渉担当者向け内部戦略メモ（外部非公開）

---

## 前提認識: スタートアップ vs 大企業の構造的不利

まず現実を直視する。

| 軸 | トヨタ | Ledra |
|---|-------|-------|
| ブランド認知 | 世界トップ3 | ほぼゼロ |
| 資本力 | 兆円規模 | スタートアップ |
| 意思決定速度 | 遅い（階層が深い） | 速い |
| 技術の複製コスト | 高い（組織変革が必要） | 中（ただし先行優位あり） |
| 代替案（BATNA） | 「自前で作る」「別のベンダー」 | **日産・ホンダ・MOBI経由** |

**不利の本質は「断られても困らない相手に売る」こと。** これを逆転するのが戦略の核心。

---

## 戦略の骨格: 3つのレバー

### レバー① 競合圧力（最重要）

> 「トヨタが動かなければ、日産・ホンダと組む」という選択肢を相手に見せる。

大企業が最も動く理由は「競合に先を越される恐怖」であり、「良い話だから採用する」ではない。

**具体的アクション:**
- **日産 PASSPORT チームに同時アプローチ**: NISSAN PASSPORT は Web3 コミュニティを既に持つ。施工証明との連携は自然。トヨタとの交渉中に「日産からも関心を持っていただいている」と言える状態を作る。
- **ホンダの NFT 特許部門にコンタクト**: Honda が 2024 年に出願した「車両情報 NFT 特許」は Ledra の施工証明と直接競合しない補完関係にある。ここへの接触は「トヨタ以外のカード」として機能する。
- **MOBI コンソーシアムに加盟**: MOBI にはトヨタ・日産・ホンダ・GM が全員参加している。Ledra が MOBI メンバーになれば、「業界標準として採用されるかもしれない技術」に変わる。トヨタ単独での囲い込みが難しくなり、逆に「今だから独占的に使える」という逆インセンティブを生み出せる。

**心理的効果:** トヨタの担当者が「うちが断ったら日産に行くのか」と感じた瞬間に、決裁スピードが上がる。

---

### レバー② データの先行蓄積（時間を味方にする）

> 施工証明が蓄積されるほど、Ledra の複製コストは上がる。今から仕込む。

ブロックチェーン上の施工記録データは **後から作り直せない**。Polygon アンカー済みの施工証明が 10 万件あるプラットフォームと、ゼロのプラットフォームは別物。これを「データモート（お堀）」として機能させる。

**具体的アクション:**
1. **アンカリングをデフォルト ON に変更**: 現在はオプションのアンカリングを、新規発行のデフォルトとして有効化。交渉時に「Polygon 上に X 万件の施工証明が既に存在する」という数字を持てる状態を作る。（`docs/vehicle-passport-design.md` の D2 参照 — アンカー率が高いほどパスポートの価値が上がる設計）
2. **車両パスポート件数を増やす**: `/v/{vin}` ページが存在する車両数が増えるほど、「Ledra を使わないと見えない車両履歴」が増える。トヨタの中古車・下取りビジネスに直結する数字として提示できる。
3. **有名施工店・ディーラー系の取り込みを先行**: トヨタ系ディーラーの 1 店でも Ledra を使っていれば、それはトヨタ社内への説得材料になる。

---

### レバー③ 「オープン標準か独占か」の選択肢設計

> 「断ったらオープン標準になる」か「今なら独占的に使える」か、どちらに転んでもトヨタが動く構造を作る。

**フレーム A: 独占先行優位を提示する**
「今 PoC を始めていただければ、向こう 18 ヶ月は他の自動車メーカーへの提供を制限します。アフターマーケット施工証明の業界標準をトヨタブランドで作る機会です。」

→ トヨタが「業界標準の設計に関与できる」というインセンティブで動く。

**フレーム B: オープン化のカードを切る**
「弊社は MOBI への加盟とオープンソース公開も検討しています。その場合、日産・ホンダも同じ基盤を使えるようになります。トヨタ独自の仕様として最初に実装いただけるなら、そちらを優先したいと考えています。」

→ 「独自仕様の機会が消える前に動け」という動機付け。

**設計の肝:** どちらのフレームを使うかは相手の反応を見て切り替える。どちらに転んでも Ledra にとっては前進。

## 交渉前に準備すべき「弾薬」

交渉テーブルに座る前に、以下の状態を作っておく。

### 数字（会議で提示できる実績値）

| 指標 | 目標状態 | 現状 |
|------|---------|------|
| Polygon アンカー済み施工証明数 | 1 万件以上 | 要確認 |
| 車両パスポート発行 VIN 数 | 1,000 VIN 以上 | 設計完了・実装中 |
| 参加施工店数 | 100 店以上 | 要確認 |
| 月次証明書発行数 | 増加トレンドを示す | 要確認 |

### 外部バリデーション

- **メディア掲載**: ASCII / ITmedia / Car Watch 等の自動車・テック媒体で Ledra の技術を取り上げてもらう（1 本でもあれば「注目されているスタートアップ」として提示できる）
- **受賞・認定**: IPA 未踏、J-Startup 認定、MOBI メンバーシップ等の第三者お墨付き
- **セキュリティ監査報告書**: 既に `docs/AUDIT_REPORT_20260329.md` がある。「エンタープライズ対応済み」の証拠として使う
- **損保会社の連携実績**: 保険会社ポータルが稼働していれば「損保 X 社が既に使っている」は強力な信用づけになる

### デモの準備

交渉相手は技術者ではなく事業開発・法務担当が多い。**「3 分で全部わかるデモ動画」** を用意する。
- シナリオ: 施工店が証明書発行 → QR スキャン → パスポートページ → Polygon Tx 確認
- トヨタ向けカスタマイズ: デモ環境の施工店名を「トヨタモビリティ ○○店」にしておく（想像させる）

## 交渉シナリオ別の対応

### パターン A: 「面白いね、でもうちは自前で作れる」

**返し:**
「おっしゃる通り、技術自体は再現可能です。ただ Polygon 上に蓄積された [X] 万件の施工証明は再現できません。また、すでに [N] 店の施工店が使っているネットワーク効果も同様です。技術を作るより、すでに動いているインフラを使う方が早く市場を押さえられると考えています。」

→ データモートとネットワーク効果を強調。「作れる」と「有利に動ける」は別。

### パターン B: 「検討します」（事実上の先送り）

**返し:**
「ありがとうございます。参考までにお聞きしたいのですが、もし 6 ヶ月後に意思決定されるとして、その時点で Ledra が日産のサービスに組み込まれていた場合、トヨタとしての立場はどう変わりますか？」

→ 「先送りのコスト」を相手自身に語らせる。

### パターン C: 「コスト（予算）が通らない」

**返し:**
「最初のステップは Woven City の施設 1 箇所でのパイロットで十分です。費用は Ledra 側で全額負担します。データと実績を共有することが目的なので、トヨタ側の金銭的コミットメントは不要です。」

→ **ゼロコスト PoC** を入口として使う。予算承認の壁を除去する。大企業の「最初の 1 歩」のハードルを下げることが最重要。

### パターン D: 「どんな契約形態を想定しているか」

**返し:**
「最初は NDA と PoC 合意書のみで問題ありません。本格展開の条件は PoC の結果を見てから一緒に設計したいと思います。」

→ 入口で複雑な契約を求めない。大企業の法務フローに引っかかる前に実績を作る。

## 社内チャンピオンの特定と育成

大企業との交渉で最も重要なのは **「社内で動いてくれる人間を作る」** こと。意思決定者を説得するより、意思決定者の周りにいる人を味方にする方が速い。

**狙うべき人物像:**
- TBL / KINTO / Woven の **中堅エンジニア・プロジェクトマネージャー**（30〜40代）
- ブロックチェーンや Web3 に個人的関心がある人
- 「新しいことをやった実績」を求めているキャリアフェーズの人

**育成方法:**
1. LinkedIn / X でトヨタ TBL・Woven 関連の発信をしている人材を特定
2. Ledra のプロダクトアップデート・技術記事を発信し、フォローされる状態を作る
3. 直接メッセージで「30 分の非公式会話」を依頼（提案書ではなく雑談として）
4. その人が「上に報告したくなる内容」を渡す（自分のポイントになる情報）

## やってはいけないこと

| NG | 理由 |
|----|------|
| 最初から独占契約・高額ライセンスを求める | 大企業の法務・調達フローで止まる。入口は無料 |
| 「うちの技術は〇〇に優れている」の自己主張だけ | 相手が気にするのは「自社ビジネスにどう役立つか」のみ |
| 1 社だけに絞って交渉する | BATNA が消え、足元を見られる |
| 担当者レベルだけとの関係で止まる | 担当者が異動すると全部リセット。複数階層に接点を持つ |
| PoC の条件を複雑にする | 始まる前に止まる。シンプルさが正義 |
| 「トヨタに採用されれば十分」という姿勢を見せる | 足元を見られる。「他にも選択肢がある」空気を維持し続ける |

## 最後に: 力関係の本質

大企業は「良いものだから採用する」ではなく、以下の理由で動く：

1. **競合に先を越される恐怖**
2. **担当者の社内評価（この話を通した人間が偉くなれるか）**
3. **経営層からのトップダウン（外圧・メディア圧力）**
4. **すでに動いているものを後から止められない（既成事実）**

Ledra の戦略は「良い技術を作る」だけでは足りない。**①競合圧力を演出し、②データを積み上げて複製コストを上げ、③ゼロコスト PoC で既成事実を作る** という 3 つを同時並行で動かすことで、「断りにくい状況」を構造的に設計する。

技術は既にある。次は「断られない状況」を作ることに集中する。
